\section{Introduction}
This document discusses the subgrammar XPPROPtoCLAUSE for Spanish.\\
The parts on rules dealing with tense and aspect have been written by 
Lisette Appelo. These are the sections on RC\_Aspect, RC\_Retro, 
TC\_Finiteness, RC\_Deixis and Superdeixisadaptation.\\

The other parts have been written by Elly van Munster.\\
Each subsection gives documentation on one Rule class or Transformation class,
and the subsections follow the order of the Control Expression as much as 
possible.



\section{Subgrammar Specification}
\begin{description}
  \item[Head] VERBPPROP, ADJPPROP, ADVPPROP, NPPROP, PREPPPROP

  \item[Import] OPENVERBPPROP, CLOSEDVERBPPROP, OPENADJPPROP, 
                CLOSEDADJPPROP, OPENADVPPROP, CLOSEDADVPPROP, OPENNPPROP,

\section{Rules and Transformations}

\begin{mruleclass}{RC\_propositionsubstitution}
\begin{classdescr}
\kind Iterative Rule Class
\classtask To substitute Propositional Phrases\\
This rule class contains the following members:
\begin{description}

\item[ROPENADJPPROPsubstitution] OPENADJPPROP as complrel
\item[RCLOSEDADJPPROPsubstitution] CLOSEDADJPPROP as complrel
\item[ROPENADVPPROPsubstitution] OPENADVPPROP as locargrel, dirargrel, complrel
\item[RCLOSEDADVPPROPsubstitution] CLOSEDADVPPROP as locargrel, dirargrel
\item[ROPENNPPROPsubstitution] OPENNPPROP as complrel
\item[RCLOSEDNPPROPsubstitution] CLOSEDNPPROP as complrel
\item[RLOPREPPPROPsubstitution] OPENPREPPPROP as locargrel
\item[RLCPREPPPROPsubstitution] CLOSEDPREPPPROP as locargrel
\item[RDOPREPPPROPsubstitution] OPENPREPPPROP as dirargrel
\item[RDCPREPPPROPsubstitution] CLOSEDPREPPPROP as dirargrel
\item[ROCPREPPPROPsubstitution] CLOSEDPREPPPROP as complrel
\item[RSENTENCEcompl] SENTENCE as complrel
\item[ROPENADJPPROPprepobj] OPENADJPPROP as prepobjrel
\item[RCLOSEDADJPPROPprepobj] CLOSEDADJPPROP as prepobjrel
\item[ROPENNPPROPprepobj] OPENNPPROP as prepobjrel
\item[RCLOSEDNPPROPprepobj] CLOSEDNPPROP as prepobjrel
\item[RSENTENCEprepobj] SENTENCE as prepobjrel
\item[RSentencesubj] `subject' SENTENCE as leftdislocrel
\item[RSENTENCEextrapos] `subject' SENTENCE as extraposrel
\item[RSentenceldisloc] `nonsubject' SENTENCE as leftdislocrel
\item[RPropsubst13] OPENVERBPPROP in ADVPPROP (Dutch "graag-like" etc.)
\item[RPropsubst14] CLOSEDVERBPPROP in ADVPPROP (Dutch "toevallig-happen" 
       etc.)
\item[Rtempadvsentsubstitution] To substitute temporal adverbial sentences
\item[Rpropsubstfuturo] Substitute a SENTENCE for the 
       complrel/SENTENCEVAR of the abstract futuro verb
\end{description}

The following rules have not been written yet for Spanish:

\begin{description}

\item CLOSEDVERBPPROP as complrel (Dutch: `hij kreeg {\em de boot afgeleverd}')
\item OPENADJPPROP as free adjunct, object antecedent 
(`Comen el carne {\em crudo}'
\item OPENADJPPROP as free adjunct, subject antecedent 
(`Duermen {\em tranquilos}')
\item OPENNPPROP as free adjunct, subject and object antecedent
(`{\em Como director} prest\'{o} atenci\'{o}n al problema')
\item OPENPREPPPROP as free adjunct, subject and object antecedent
(`Ley\'{o} el libro {\em con ojos cansados}')
\item CLOSEDADJPPROP as free resultative adjunct to intransitive and 
pseudo-transitive activity verbs (Dutch: `hij dronk zijn glas {\em leeg}')
\item CLOSEDNPPROP as free resultative adjunct, though this seems to be possible
in idiomatic cases only (Dutch: `Hij schrok {\em zich een hoedje}')
\item Substitution of adverbial sentences (Dutch: Rconjsentsubst1, 
Rconjsentsubst2, Rconjsentsubst3, Rconjsentsubst4)
\item Substitution of sentences in ADJPPROPs (Dutch: Rsadjsentsubst)

\item  {\bf. NB} A substitution rule for OPENVERBPPROPs as complrel is 
unnecessary.

\end{description}
\end{classdescr}

See also doc.\ 53 and 263.\\
Note that the attribute aspect and its values have spanish names in the rules:\
\
\begin{tabular}{ll}
aspect: & {\em aspecto}\\
imperfective:& {\em imperfectivo}\\
perfective:& {\em perfectivo}\\
\end{tabular}
\\
In the documentation, the English names will be used most of the time.

\end{classdescr}

\begin{members}
\begin{member}
\rulename Raspectimperfective

forms later) or left {\em omegaaspecto} (other tense forms).

reference interval R

\example x$_{1}$ trabajar$_{omegaaspecto}$ $\rightarrow$ 
x$_{1}$ trabajar$_{imperfectivo}$\\ 
x$_{1}$ trabajar$_{omegaaspecto}$ $\rightarrow$ 
x$_{1}$ trabajar$_{omegaaspecto}$

\begin{mruleclass}{RC\_deixis}
\begin{classdescr}
\kind Obligatory Rule Class
\classtask To set the attributes {\bf deixis} and {\bf superdeixis} of the 
CLAUSE at the appropriate values, and to spell out the tense of the first verb  
for finite clauses.\\
These rules establish the relation between the reference interval Rs and S.\\
The rule class contains parallel rules for deixis and superdeixis of the CLAUSE 
since at can be dependent of a higher clause for its time (superdeixis) or not 
(deixis). First the deixis rules will be tyreated (A), then the superdeixis 
rules (B).\\
Note that the Spanish rules contain a Spanish name for {\em deixis} and {\em 
tense} and their values: \\
\begin{tabular}{ll}
deixis: & {\em tiempo}\\
tense: & {\em tiempo}\\
superdeixis: & {\em supertiempo}\\
presentdeixis:& {\em presente}\\
pastdeixis: & {\em pasado}\\
omegadeixis:&  {\em omegatiempo}\\
presenttense: & {\em presente}\\
pasttense: & {\em pasado}\\
omegatense: & {\em omegatiempo}\\
\end{tabular}
\\
The documentation often uses the English indications!!\\
See also doc.\ 53 and 263 (on temporal expressions in Rosetta3) and doc.\ 320 
(on superdeixis in Rosetta3).
\end{classdescr}

\subsubsection{A. Deixis rules}
\begin{members}
\begin{member}
\rulename Rfinpresentdeixis
\ruletask                This rule is for a finite independent clause.
               The most left verb in the clause gets a present tense ({\em 
presente}).
               The deixis 
value of the reference adverbial is checked for 
               present.
               The clause is marked for present deixis ({\em presente}).
\file spanish:RC\_deixis.mrule (mrules39.mrule)
\semantics The relation between Rs and S is simultanuous: PRESENT.

\example duerme
\remarks\mbox{}
The superdeixis value also may be presentdeixis in the case 
this sentence is a sentence with adverbs such as {\em graag} in Dutch. 
In Spanish this may be 
the case in cases in which e.g. a verb is a semantic head but behaves as an  
auxiliary verb (e.g. some perifrasis verbales or may be modal verbs)\\
Conditions on adverbials have been added to avoid the 
perfective reading of: {\em Nada por tres horas}.\\
In case of {\em temporally independent} but {\em syntactically dependent} 
clause only a presentdeixis referential adverbial is allowed. This condition 
will be taken care of in the superdeixis adaptation transformations. With a 
neutral referential time expression such a clause will always be interpreted as 
{\em temporally dependent} and will be treated in RfinPresentSuperdeixis.

\end{member}

\begin{member}
\rulename Rfinpresfutdeixis
\ruletask                This rule is for a finite independent clause.
               The most left verb in the clause gets a present future tense
({\em present, sifuturo}).
               The deixis 
value of the reference adverbial is checked for 
               present, future or omega.
               The clause is marked for present deixis ({\em presente}).
\file spanish:TC\_futuro.mrule (mrules44.mrule)
\semantics The relation between Rs and S is simultanuous: PRESENT, but with 
S `shifted' into the future.

\example dormir\'{a} ma\~{n}ana, dormir\'{a} a las tres, dormir\'{a} 
hoy por tres horas.
\remarks\mbox{}
The superdeixis value also may be presentdeixis in the case 
this sentence is a sentence with adverbs such as {\em graag} in Dutch. 
In Spanish this may be 
the case in cases in which e.g. a verb is a semantic head but behaves as an  
auxiliary verb (e.g. some perifrasis verbales or may be modal verbs)\\
There are conditions for a present adverbial in 
combination with perfective aspect and a duration adverbial. These cases are 
excluded from the present deixis rule (Rfinpresentdeixis).\\ 
Note that there is another interpretation of the {\em futuro}, corresponding to 
the Dutch verb {\em zullen}, which is treated as a modal verb. In Spanish the 
abstract verb {\em futuro} has been introduced for these cases. The abstract 
verb is excluded from this rule. The rule 
Rfinfutpresdeixis accounts for the present deixis in that case.\\
See for the treatment of modal verbs in Rosetta3 doc.\ 327.

\end{member}

\begin{mruleclass}{TC\_superdeixisadaptation}
\begin{classdescr}
\kind Obligatory Transformation Class
\classtask To check the superdeixis of the embedded 
propositional phrases and sentences with the (super)deixis 
of the clause and to adapt superdeixis (and deixis)
of embedded propositional phrases 
and clauses on behalf of the surface parser.\\
There are two kinds of superdeixis adaptation transformations: non-iterative 
and iterative. The non-iterative ones
form an {\em obligatory} transformation class 
with a {\em default} transformation which is the negation of all other 
transformations of that class. This class is followed by several 
iterative superdeixis adaptation transformation (classes) each with a filter.
This means that the default transformation of the obligatory non-iterative 
class also must let pass the iterative cases, but of course, these cases are not 
formulated as the negation of the iterative rules!!\\
There are in general 5 subcases for the embedded propositional phrase or 
sentence in every transformation or subrule of a 
transformation, although they do not apply always in every transformation.

\begin{enumerate}
\item {\bf independent }:\\
No check, no adaptation.\\
It is checked if the verb of the CLAUSE can have independent complements
(AUX\_tempindepcomplvps).\\
(Only applies to finite sentences and a special case of infinite sentences)
\item {\bf dependent finite present }:\\
Check + adaptation of deixis and superdeixis.\\
(Only applies to sentences)
\item {\bf dependent finite past}:\\
Check + adaptation of deixis and superdeixis.\\
(Only applies to sentences)
\item {\bf dependent infinite present}:\\
Check + adaptation of superdeixis.
\item {\bf dependent infinite past}:\\
Check + adaptation of superdeixis.

\end{enumerate}
In analysis these transformation cause a lot of ambiguity. Therefore, 
efficiency and Rosetta3-plausibility conditions have been added to these 
transformations. 
\begin{itemize}
\item Tests for the presence of adverbials are added to cut off some possible 
paths here instead of in the deixis rules 
(QUOTE\_temprefnotfound). The test for the correct adverbial is 
left to the deixis rules. 
\item It seemed plausible that only infinite sentences that contain {\em 
haber} \'{a}nd an overt reference time adverbial can be independent.
For this purpose the transformations for {\em embedded} sentences 
have been split into two subrules: one that tests for the presence of {\em 
haber} and one that tests for the absence of {\em haber}.
\item The combinations present (super)deixis of the clause with present deixis 
(i.e.\ independent!) without an overt adverbial in the embedded sentence and 
past 
(super)deixis of the clause with past deixis (i.e.\ independent!) without an 
overt adverbial in the embedded sentence are not considered plausible in an 
application as Rosetta3 and ruled out in these transformations.
\end{itemize}
The motivation and historical reasons for the (actual form of the) 
transformations can be found in doc.\ R263 and a description of the treatment 
of superdeixis through the whole grammar in doc.\ R320.\\
The two files that contain the superdeixis adpatation transformations are 
organized roughly in the following way: tc\_superdeixisadaptation contains the 
transformations with embedded {\em sentences} and tc\_superdeixisadaptation2 
the ones with embedded {\em xpprops}and {\em xps}.\\
First the non-iterative obligatory rule class (A) will be treated and then the 
iterative cases (B).
\end{classdescr}

\end{member}
\begin{member}
\rulename TverbRaising
\ruletask To form Verb raising structures.
\file spanish:TC\_sentenceOK.mrule (mrules26.mrule)
\semantics \nosemantics
\example puede [hacer-lo] $\rightarrow$ [puede hacer] lo [ ]
\remarks\mbox{}
\item problems:\\
Since at this moment only subject-control verbs allow Clustering, i.e. 
only verbs without there own (clitic)arguments,
the rule can be easy: the CLITICnode in the complrel just
moves to the right of VERBcluster-node under the CLAUSE.
The CLITIC-node of the nain clause (which must be empty) is deleted.\\
In the future, if also verbs with there own arguments allow Clustering, the 
rule will have to be more complicated, since the two CLITICnodes will have to 
merge. E.g.: \\
Me permite hacerlo $\rightarrow$ Me lo permite hacer.\\
If the verb is a subject-controller, the subject of main- and embedded sentence
is identical per definition, so the rule doesn't have to test this.\\
Again, if also (ind)object-controllers are added as verbraiser, the rule has to 
test if in these cases the (ind)obj of the main sentence is identical to the 
subject of the embedded sentence. \\
This rule is for finite clauses. If it should also be for infinite clauses, 
there are two options: 
\begin{enumerate}
\item The glue rules are sensitive for (very deep) embedded infinitives in verb 
clusters
\item This rule is extended with the possibility to put the glue + clitic into 
the verb cluster.\\
\end{enumerate}

It is assumed that verbs in spanish are never oblVR except for {\em haber}. If 
such verbs would exist, this rule should allow for the absence of a CLITIC node 
in the embedded sentence. Now, there is always the path of noclustering and 
without clitics it is not possible to see if the verb is raised or not.\\
This rule supposes that the clitic in the embedded snetence is to the right of 
all the verbs. An exception to this is {\em haberlo hecho}. Then the clitic is 
between the auxiliary verb and the main verb. We need an extra rule for this 
possibility as the model is different.\\

\item modifications:
\begin{itemize}
\item 9/11/88: added glue to input model; this is necessary as in the 
cycle of the sentence such a glue is put obligatorily between infinitives or 
gerundios and the clitics. 
\end{itemize}

\end{member}

\begin{mruleclass}{TC\_control}
\begin{classdescr}
\kind Obligatory Transformation Class
\classtask (copied from Dutch doc.nr.314)\\
To deal with control (the interpretation of nonovert subjects of
infinitival sentences) and to deal with the interpretation of nonovert subjects
of OPENXPPROPs. It also adapts the structure of OPENXPPROPs and CLOSEDXPPROPs,
turning OPENXPPROPs into XPs and CLOSEDXPPROPs into objrel/NP(VAR) + XP.
\end{classdescr}

\begin{description}
   \item[TControl0] for CLAUSEs without OPENXPPROPs, CLOSEDXPPROPs or
                    infinitival SENTENCES.
   \item[TOblControl1a] OPENADJPPPROP, object antecedent
   \item[TOblControl1b] OPENADJPPPROP, subject antecedent
   \item[TNoControl1]   CLOSEDADJPPROP
   \item[TOblControl2a] OPENADJPPPROP in prepobjrel/.., object antecedent
   \item[TOblControl2b] OPENADJPPPROP in prepobjrel/.., subject antecedent
   \item[TNoControl2]   CLOSEDADJPPROP in prepobjrel/..
   \item[TOblControl3a] OPENNPPROP, object antecedent, idrel and predrel
   \item[TOblControl3b]  OPENNPPROP, subject antecedent, idrel and predrel
   \item[TNoControl3] CLOSEDNPPROP, predrel
   \item[TNoControl3id] CLOSEDNPPROP, idrel\\
   \item[TOblcontrol7a] OPENADVPPROP, object antecedent, locargrel, dirargrel,
                                                         complrel
   \item[TOblcontrol7b] OPENADVPPROP, subject antecedent, locargrel, dirargrel,
                                                          complrel
   \item[TNocontrol7] CLOSEDADVPPROP, objrel (Dutch: locargrel, dirargrel! 
                      Maybe it has to be changed for Spanish)
   \item[TOblControl4a] OPENNPPROP, object antecedent, idrel and predrel, in
                         prepobjrel/..
   \item[TOblControl4b] OPENNPPROP, subject antecedent, idrel and predrel, in
                         prepobjrel/..
   \item[TNoControl4] CLOSEDNPPROP, predrel. in prepobjrel/..
   \item[TNoControl4id] CLOSEDNPPROP, idrel. in prepobjrel/..
   \item[TOblControl5a] OPENPREPPPROP, object antecedent (VARPREPP)
   \item[TOblControl5b] OPENPREPPPROP, subject antecedent  (VARPREPP)

   \item[TOblControl6a] SENTENCE, nonsubject antecedent
   \item[TOblControl6b] SENTENCE, subject antecedent

\begin{mruleclass}{RC\_emptysubst}
\begin{classdescr}
\kind Iterative Rule Class
\classtask Substitution of EMPTYs.\\

\begin{itemize}
\item We assume there are four different EMPTY's:
\begin{enumerate}
\item seEMPTY (Dutch "men")
\item ellosEMPTY (Dutch "ze generic")
\item emptyEMPTY (Dutch "empty argument")
\item unoEMPTY (Dutch "je generic")
\end{enumerate}
\item The following EMPTYsubst-rules have been made:
\begin{enumerate}
\item seEMPTY in a por-phrase (Active and SEactive).
\item emptyEMPTY as objrel (Active).
\item emptyEMPTY as indobjrel (Active).
\item seEMPTY as subjrel (SEactive).
\item ellosEMPTY as subjrel (SEactive).
\item seEMPTY as subjrel $\rightarrow$ {\bf UNO} with refl.V. (Dutch "men").
\item seEMPTY as subjrel $\rightarrow$ {\bf UNO} with non-refl.V. (Dutch "men").

This rule has a very low bonus but is necessary for cases where later on the 
derivation the occurrence of a second SE blocks rule 4.\\
(-*Se se lava las manos $\rightarrow$ {\em uno} se lava las manos).
\item unoEMPTY as subjrel (Dutch "je generic").
\item seEMPTY as indobjrel (Dutch vp120 with "men" as subj into Spanish 
vp012 with EMPTYVAR in indobj-position, translated into "les..a la gente"; 
e.g.:\\
- Men houdt van kaas. $\rightarrow$ Les gusta el queso a la gente.
\item seEMPTY as objrel (SEactive, ergative verb, vp010).
\end{enumerate}
\item The following translations D-Sp are obtained:\\
- Men danst $\rightarrow$ \\ - Se baila \\
- Er wordt gedanst $\rightarrow$ \\ - Se baila \\
- Men schaamt zich $\rightarrow$ \\ - Uno se averg\"{u}enza \\
- Men wast z'n handen $\rightarrow$ \\ - Uno se lava las manos \\
- Ze dansen $\rightarrow$ \\ - Bailan \\
- Er worden appels gegeten $\rightarrow$ \\ - Se comen manzanas \\

  chicos \\

- Er worden kinderen (generic!) gezocht $\rightarrow$ \\ - Se buscan ni\~{n}os \\
- Het boek wordt door Jan gelezen $\rightarrow$ \\ - Juan lee el libro \\
  - El libro es le\'{\i}do por Juan 
\item The porobjrel/.. is VERBP-final, in order to avoid two adjacent mu's, 
which would make the rule much more complicated.
\end{itemize}

\begin{mruleclass}{TC\_objectOK}
\begin{classdescr}
\kind Obligatory Transformation Class
\classtask To guarantee that there is going to be a subject in the sentence if 
required.\\

The following ObjectOK-rules have been made:
\begin{itemize}
\item TobjectOK0:\\

defaultrule (VERB = not-passive and not-ergative).\\
(NB. Also for existentials !!)

\item TobjectOK1a:\\

VERB = passive and not-ergative.\\
object = definite.\\
object becomes subject.

\item TobjectOK1b:\\

VERB = passive and not-ergative.\\
object = non-definite.\\
object becomes subject.

\item TobjectOK2a:\\

VERB = active and ergative.\\
object = definite.\\
object becomes subject.

\item TobjectOK2b:\\

VERB = active and ergative.\\
object = non-definite.\\
object becomes subject.

\item TobjectOK2c:\\

VERB = active and ergative.\\
object = SE.\\
object becomes subject.

\item TobjectOK3a:\\

VERB = passive and not-ergative.\\
object = definite.\\
object remains object.

\item TobjectOK3b:\\

VERB = passive and not-ergative.\\
object = not-definite.\\
object remains object.

\item TobjectOK4a:\\

VERB = not-passive and ergative (vp010 !).\\
object = definite.\\
object remains object.

\item TobjectOK4b:\\

VERB = not-passive and ergative (vp010 !).\\
object = non-definite.\\
object remains object.

\item TobjectOK5:\\

VERB = SEpassive and vp120/vp123.\\
object remains object.\\
Introduction of subjrel/PRONP[EC].\\
Also for "hay" (existential).

\item TobjectOK6a:\\

VERB = passive or ergative.\\
"subject-to-subject raising" (of NP and PRONPVAR).\\
complrel/SENTENCE.

\item TobjectOK6b:\\

VERB = passive or ergative.\\
"subject-to-subject raising" (of NP and PRONPVAR).\\
complrel/PREPP..[SENTENCE].

\item TobjectOK7:\\

VERB = ergative.\\
complrel/SENTENCE.\\
No "subject-to-subject raising" possible.
introduction of "redello".

\item TobjectOK8:\\

VERB = ergative.\\
complrel/SENTENCE (extraposed).\\
No "subject-to-subject raising" possible.\\
introduction of "redello".
\end{itemize}



NB. I made the splitting up into rules for {\em def} vs. {\em non-def} 
objects, in order to be able to give a different bonus to these rules.\\
In principle, both def and non-def objects can be either in subject or
in object position in Spanish, but the degree of acceptability is different.\\
e.g. (1) is more natural than (2) although both are correct:\\
(1) - Llegaron veinte soldados. \\
(2) - Veinte soldados llegaron. \\
Note that not only the (non)definiteness of the object is a determining factor,
but also the type of verb (llegar = a "presentational" verb)

\end{classdescr}
