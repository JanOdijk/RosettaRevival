\section{ RC:Clause-formation}

\begin{description} 
  \item[HEADCATS] VERBPPROP, ADJPPROP, ADVPPROP, NPPROP, PREPPPROP
  \item[IMPORTCATS]To be specified by Lisette Appelo
  \item[EXPORTCATS] CLAUSE, VERBPFORMULA, ADJPFORMULA, ADVPFORMULA, NPFORMULA,
                   PREPPFORMULA
\end{description}

This is in fact not a rule class, but a package of rules and transformations
that are specified further by Lisette Appelo (cf. doc 0054)

 These rules turn the VERBPPROP into a CLAUSE, introduce time and aspect
and spell these out by means of changing attribute values and introducing 
auxiliary verbs.


In Dutch e.g. the auxiliaries {\em hebben} or {\em zijn} are introduced to spell
out perfect aspect. They are put to the right of VERBP with relation 
{\em auxrel}.
Auxiliary combinations such as {\em aan het Vinf zijn} can be introduced here
Its structure must be determined more precisely.

The passive auxiliaries {\em worden} and {\em zijn} are spelled out here
These are put to the right of VERBP under relation {\em  auxrel}

 If the predrel is an ADJP, NP, PREPP or ADVP, the copula {\em zijn} is 
introduced, or if necessary some other copula (perhaps {\em  liggen, staan}).


In XPPROPtoXPFORMULA subgrammars there is a default rule that has the same
meaning as the rule that introduces dependent deixis, finite or infinitival. 
In the XPPROPtoXPFORMULA subgrammars they effectively do nothing, except that
a category change from XPPROP to XPFORMULA is achieved.

\subsection{ RC:Clause-formation in XPPROPTOCLAUSE}

This rule spells out time and aspect by introducing auxiliary verbs and setting
certain values for certain attributes. It changes the topcategory from
XPPROP into CLAUSE. In the case of adjectives for present en past tense in 
Dutch the auxiliary verb {\em zijn} is introduced.

Copular verbs like {\em lijken}, {\em blijken} etc. are not introduced here,
but are basic expressions themselves. The same holds for such verbs as
{\em worden } or e.g. {\em raken } as in {\em  hij raakte dronken}.

\section{TC:Conjunctiondeletion}

\begin{description} 
  \item[HEADCATS] SENTENCE
  \item[IMPORTCATS] --
  \item[EXPORTCATS] SENTENCE
\end{description}

In this transformation class certain conjunction are deleted, in particular
{\em of } in subordinate wh-interrogative questions (e.g. {\em wie of er kwam }
$\rightarrow$ {\em wie er kwam }. The {\em of } must be present in such cases
to avoid the application of V/2 here. The conjunction {\em om } cannot be 
deleted here, since its appearance is co-determined by external factors, e.g.
deletion is possible iff the sentence is a complement of verbs, otherwise it is
not.

\section{ TC: ErHopping, FC:ErHopping}

\begin{description} 
  \item[HEADCATS] CLAUSE
  \item[IMPORTCATS] --
  \item[EXPORTCATS] CLAUSE
\end{description}

This transformation class hops RADVs out of a PREPP to a position
to the left where they are going to bear the relation ERPOSREL

In Rosetta2, and also in Domain T of Rosetta3 it is assumed that,
if the RADV is hopped out of the PREPP then the PREPP node disappears
and the remaining PREP-s are going to bear a special relation, 
i.c. {\em strandedrel} (R3).

I propose not to do this, cf. e.g. {\em er gisteren vlak voor }, where there
are two elements in the PREPP. I see no objections for not doing it (e.g.
in the surface parser), but if there are any, then we should perhaps reconsider
it again.

The relation {\em strandedrel } might still be useful, for such cases as
{\em er geen kaas {\bf van } gegeten hebben }, where the PREP {\em van } is
not in its normal position. See below.

Prepositions in PREPPs where HETtoER applies get the value {\em true } for the
attribute {\em postform} (cf. Angelieks scriptie), so that the morphology can
achieve the required changes (e.g. {\em  met} $\rightarrow$ {\em mee}, {\em
tot} $\rightarrow$ {\em toe} ). 

The PREPP from which the hopping takes place must be
\begin{itemize}
  \item adjacent to the VERB  (cf. {\em ..*dat hij er toch {\em naar} niet graag
{\em kijkt} }
   vs.  {\em  ..dat hij er toch niet graag {\em naar} {\em kijkt}}
  \item adjacent to a directional or locational PREPP which is adjacent to the
verb (in particular: {\em met NP }, cf. {\em het ijsje, waar hij mee op het hek 
zit }). 
  \item adjacent to an NP that is an idiom part and  that is adjacent to V  
(e.g. {\em van iets {\em geen kaas} {\em gegeten} hebben}) 
  \item adjacent to a resultative adjective 
  ( e.g. {\em  iets met iets {\em groen} {\em verven}})
\end{itemize}


In the latter two cases the prepositions move over these constituents, cf.
{\em  er geen kaas {\bf van} gegeten hebben}, {\em het hek er groen {\bf mee}
geverfd hebben } etc.

This movement must apply also in cases of {\em ergens}, {\em hier} etc. where
the hop is not obligatory! 

Cf. also {\em Hij heeft er iets mee naar toe gebracht/genomen} vs. 
            {\em  iets naar iets toe mee brengen/nemen}

 (cf. e.g. Angeliek's scriptie. p. 110-111) 

Er-hopping is obligatory for {\em er}, optional for all other pronouns.

No form can be shifted directly from the PREPP, so that all forms that
must be shifted anyway must first hop to the position where they are going to
bear the relation ERPOSREL.

Maybe certain PREPP-s must be shifted via ERPOSREL as well, cf. e.g. 
{\em  daarop werd (*er) geantwoord; de vraag, waarop (*er) werd geantwoord}
vs. {\em de manier waarop (er) werd geantwoord }.

Also relevant in this connection are
\begin{enumerate}
  \item[] In de kast lagen (*er\{x\}) twee boeken 
  \item[] In de tuin werd er gedanst              
  \item[] Op deze vraag werd (*er) niet geantwoord
\end{enumerate}
These examples seeem to indicate that all (non question) argument PREPP's that
are shifted go via ERPOSREL. 

I assume, following Angeliek, that RADVs have an attribute, which
I call  {\em kinds} (Angeliek called it {\em kind}), that can have as its value
a set of ( L, P, X, Q ). The values containing {\em X} and {\em Q} are relevant
only to {\em er}. Sets containing both {\em L} and {\em  P} are illegal. This
yields the following possible sets for 'er': 
\begin{itemize}
  \item \{L\}, \{P\}, \{X\}, \{Q\}, \{L,X\}, \{L,Q\}, \{P,X\}, \{P, Q\}, \{X,
Q\}, \{L, X, Q\}, \{ P,X,Q\} (no.: 11) 
\end{itemize}

 and for other RADVs:

\begin{itemize}
  \item \{L\}, \{P\}
\end{itemize}

examples of the combinations for {\em er } are:
\begin{description}
   \item[\{L\}]  Hij woont {\em  er}
   \item[\{P\}] Hij kijkt {\em er} nu naar
   \item[ \{X\}] {\em Er } kwam een jongen binnen
   \item[ \{Q\}] Ik zag {\em er } daar twee
   \item[ \{L, X\}] {\em  Er } woont iemand
   \item[ \{L, Q\}] ik zag {\em er } twee
   \item[ \{P, X\}] {\em Er } keek iemand naar
   \item[ \{P, Q\}] Ik las {\em er } twee over
   \item[ \{X, Q\}] .. dat {\em er } daar twee zijn
   \item[ \{L, X, Q\}] .. dat {\em er } twee liggen
   \item[ \{P, X, Q\}] .. dat {\em er} twee in stonden  (?. ..dat {\em er} er
twee in stonden) 
\end{description}

The values that contain {\em Q} can only come into existence by the 
fusion of two {\em er}s into one by TC:erqerdeletion.
 
We might also consider defining these sets as primitive values, e.g.
the values {\em  L, P, X, Q, LX, LQ, PX, PQ, XQ, LXQ} but this will probably
complicate the formulation of certain rules slightly.

{\em In contrast to what has been assumed in Domain T, I assume that {\bf
kindefs} is not a \{EFF\}SET OF kindtype, but an \{EFF\}SET OF SET OF kindtype.
Domain T should be changed accordingly }

The target position of the hopping is a position to the left of VERBP,
(or PREPP, ADJP etc. in case of copular sentences), bearing the relation
ERPOSREL. At the moment of the hopping this position can be filled already
by ER \{x\}, \{l\}, \{p\}, \{x,l\} or \{x,p\}, or by some other RADV \{p\}.
The \{p\} cases can exist due to the fact that this transformation class is
iterative. (See below). The following scheme indicates the actions to be
taken:

\begin{tabbing}
aaaaaaaaaaaaaaa \= aaaaaaaaaaaaaaaaaaaa \= aaaaaaaaaaaaaaaaaaaaaaaaaa \kill

RPOSREL         \> HOPPING RADV          \> result:\\

not present     \> any, say T1             \> rposrel/T1\{p\}\\
ER\{x\}         \> ER                      \> rposrel/ER\{p,x\}\\
                \> any other RADV      \> rposrel/RADV\{p\}\\
ER\{l\}         \> any                     \> deriv. blocked\\
ER\{p\}         \> ER                      \> rposrel/ER\{p\}\\
                \> any other RADV      \> deriv. blocked\\
ER\{x,l\}       \> any                     \> deriv. blocked\\
ER\{x,p\}       \> ER                      \> rposrel/ER\{x,p\}\\
                \> any other RADV      \> deriv. blocked\\
RADV\{p\}   \> the same RADV       \> rposrel/RADV\{p\}\\ 
\end{tabbing}

If the hopping element is a VAR, then VAR behaves as e.g. {\em daar} does.

The transformation class must be iterative because more than one PP that
must be subject to this rule can occur in the structure. Since the relevant
operations are obligatory, at least for some cases (e.g. if the hopping element
is {\em ER}) a filter transformation must follow this transformation class.

If no Erhopping is applied, then a GLUE is inserted between {\em  er, daar, 
hier, RADVVAR} and the preposition.
Notice that, given this, we cannot generate {\em we hebben ernaar gekeken },
but only {\em we hebben er naar gekeken }, since in this sentence {\em er }
must hop obligatorily. I assume that a later transformation adds a GLUE
between {\em er,daar, hier } in erposrel and a stranded preposition, but
this will also yield ungrammatical (i.e. violating orthographic conventions)
results, so perhaps we should do so only
in analysis (cf.?*{\em we zijn ermee naartoe gegaan } vs. {\em we zijn er mee 
naartoe gegaan }. What about {\em er naar toe, ?ernaartoe, ?ernaar toe, 
?ernaar toe, er naartoe}. This transformation must be rather late. {\em Its
existence is not indicated anywhere yet in the subgrammar definitions}.

\section{ TC: ERQERdeletion}

\begin{description} 
  \item[HEADCATS] SENTENCE
  \item[IMPORTCATS]--
  \item[EXPORTCATS] SENTENCE
\end{description}

This transformation class transforms the structure

\begin{itemize}
  \item ...erqrel/ER1 erposrel/ER2 ...
\end{itemize}

 into
\begin{itemize}
  \item ... erposrel/ER3 ..
\end{itemize}

 where ER3.kinds = ER1.kinds + ER2.kinds

 Example:

\begin{itemize}
  \item ...dat er\{q\} er\{x\} twee geschreven zijn  $\rightarrow$
  \item ...dat er\{q,x\}     twee geschreven zijn
\end{itemize}

The transformation class is crucially ordered {\em after} TC:Verbsecond

\section{ TC:HettoEr, FC:HettoEr }

\begin{description} 
  \item[HEADCATS] CLAUSE, ADJPFORMULA, ADVPFORMULA, NPFORMULA, PREPPFORMULA,
                  VERBPFORMULA, NP
  \item[IMPORTCATS]--
  \item[EXPORTCATS]  CLAUSE, ADJPFORMULA, ADVPFORMULA, NPFORMULA, PREPPFORMULA,
                  VERBPFORMULA, NP
\end{description}

 This transformation class accounts for the fact that {\em het} after
prepositions is turned into {\em er}. Summarized: 
\begin{description} 
  \item[het]   $\rightarrow$ er
  \item[dit]   $\rightarrow$ hier
  \item[dat]   $\rightarrow$ daar
  \item[iets]  $\rightarrow$ ergens   (optionally)
  \item[alles] $\rightarrow$ overal   (optionally)
  \item[niets] $\rightarrow$ nergens  (optionally)
  \item[wat]   $\rightarrow$ waar
\end{description}

Notice that {\em wat $\rightarrow$ waar} is in most cases not applicable here, since
it has not been inserted yet. Instead the variable that is present must change.

In contrast to what has been assumed in earlier stages, I will assume that
this change is really made here by choosing different basic categories as in 
Rosetta2,  and not by assigning the value true to the attribute R in these
words. 

The R-words are positioned on the left in the PREPP:
\begin{verbatim}

    PREPP                          PREPP

head    obj                    errel        head

PREP    NP         -->         RADV         PREP

        head

 DEMPRO/PERSPRO/INDEFPRO 

\end{verbatim}

If a degree modifier is present, then the RADV is put behind it
(cf. {\em  *Er vlak voor stond een vuilnisbak} vs. {\em Vlak ervoor stond een
vuilnisbak}). 
In certain cases the string {\em er vlak voor} can be correct, but then (at
least it looks as if) {\em  er} has been placed out of the PREPP. 

If the object is a NPVAR that is going to be substituted by {\em wat} (we must
know this by the pre-action), then this NPVAR is also put to the left, replaced
by an RADVVAR with the same index and  {\em \{mood:wh, kindefs:[P]\}}
and it starts to bear the relation {\em errel}. {\em Is this possible?}


If the PREPP contains a postposition, the change {\em  het}$\rightarrow$ {\em  
er} does not apply.
( Cf. {\em Hij is er in gelopen} does not mean: {\em hij is het bos in gelopen}
,cf., however, also {\em ?hij is er in gereden, ?hij is er in gegaan },
de garage, waar hij in is gereden)

In the case of circumpositions the RADV is attached to the highest PREPP:

\begin{verbatim}

       PREPP1                                         PREPP1

   obj          head                         errel    p1rel     head    

   PREPP2       PREP2                -->     RADV     PREP1     PREP2

head    obj     door                          daar    onder    door


PREP1   DEMPRO

onder    dat

\end{verbatim}

The rules decribed here apply in all XPFORMULAtoXPPROP subgrammars and
in the CLAUSEtoSENTENCE subgrammar. This is necessary to account for
both 'regular PREPP-s' and for prepobj/PREPP-s (which are not formed by
the PREPPPROPformation subgrammar)

 The transformation class must be applied iteratively (at least in 
the clausetosentence subgrammar, since more than one PP can occur in the
structure, and a filter class must follow it to guarantee that the rule is
applied indeed in certain cases.

\section{ RC:MakeAdvsent}

\begin{description} 
  \item[HEADCATS] PREP, CONJ
  \item[IMPORTCATS] SENTENCE
  \item[EXPORTCATS] ADVSENT, PREPP
\end{description}


Special rules to form adverbial sentences. These rules combine 
adverbial subordinate conjunctions or prepositions with clauses.
They belong in the ADVSENT subgrammar.

One might also consider the possibility of making such sentences in the 
mooddetermination rules. A disadvantage of the approach adopted here is that
certain effects of transformations must be achieved here by writing
separate rules. An example:
The conj {\em als} + CLAUSE can yield a SENTENCE with {\em als} as its
conjunction, or it can yield a sentence without a conjunction: 
\begin{itemize}
  \item {\em  als} + CLAUSE[ hij komt] $\rightarrow$ SENTENCE[conjrel/als hij
komt ] 
  \item {\em als} + CLAUSE[ hij komt] $\rightarrow$ SENTENCE[ hij komt ]
\end{itemize}
             
In the first case the adverbial subordinate sentence {\em  als hij komt}
results; in the second case the absence of a conjrel/.. will trigger the 
application of Verbsecond, yielding the adverbial subordinate sentence 
{\em  komt hij} ( as in {\em  Komt hij, dan kunnen we dit gaan doen}).
  
If the formation of adverbial clauses takes place in a special subgrammar,
as done here, then the effect of Verbsecond in such contexts must be achieved
by a special rule. 

The rules combine a.o.
\begin{itemize}
  \item prepositions such as {\em zonder}, {\em buiten}, {\em gegeven}, {\em
behalve } with finite declarative sentences to form  PREPP-s; 
 In cases such as {\em voordat}, {\em nadat}, {\em doordat} a GLUE should be
introduced, or they should be analyzed as CONJ-s.
  \item conjunctions such as {\em terwijl}, {\em als}, {\em toen}, {\em zodra}
with finite declarative sentences to form finite ADVSENT-s 
  \item prepositions such as {\em zonder}, {\em door}, {\em in plaats van} etc.
with declarative te-infsentences to form PREPP-s.
   \item conjunctions such as {\em alvorens}, {\em teneinde} with infinitival
declarative sentences to form infinitival ADVSENT-s. 
etc.
\end{itemize}

 Examples:
\begin{enumerate}
  \item[] ..zonder dat hij het wist ; .. behalve dat er geen voedsel was
  \item[] ..als we dat doen; ..terwijl hij hier mee bezig is 
  \item[] ..zonder om zich heen te kijken ; ..door te vertrekken
  \item[] ..alvorens te vertrekken ; teneinde dit te bereiken
\end{enumerate}

A special case is formed by adverbial sentences formed by {\em zoals }.
Consider the following sentences:

\begin{enumerate}
  \item[] Zoals u ziet ontbreekt er een argument in deze zin na {\em ziet}
  \item[] Zoals u zult begrijpen, moet hier iets speciaals voor gedaan worden
  \item[]*Zoals u eet, is deze zin wel erg raar
  \item[] Zoals u zult moeten kunnen proberen in te zien, kan het ontbrekende
argument erg 'diep' staan.
  \item[] Zoals nu duidelijk moet zijn, heeft hier shift plaats gevonden
\end{enumerate}

Adverbial sentences introduced by {\em zoals} (English: {\em  as}) lack
one argument. This must be an argument that, if present, can be a sentence.
Hence the third example given is ill-formed, since {\em eten } does not take a sentential complement.
The main sentence is substituted for the shifted variable, resulting into
a sentence where the main sentence contains the {\em zoals-}sentence and the
shifted SENTENCEVAR has been deleted.

Notice by the way that shifting of SENTENCEVARs is quite special an operation.

\section{ RC: Mooddetermination}

\begin{description} 
  \item[HEADCATS] CLAUSE, VERBPFORMULA, ADJPFORMULA, ADVPFORMULA,NPFORMULA,
                  PREPPFORMULA
  \item[IMPORTCATS]--
  \item[EXPORTCATS] SENTENCE, OPENVERBPPROP, OPENADJPPROP, OPENADVPPROP,
         OPENNPPROP, OPENPREPPPROP, CLOSEDVERBPPROP, CLOSEDADJPPROP,
         CLOSEDADVPPROP, CLOSEDNPPROP, CLOSEDPREPPPROP 
\end{description}


This rule class sets the mood of a sentence, and spells out conjunctions
if these are required. This differs from earlier versions of the iso-scheme,
where the conjunctions were spelled out by a separate transformation

A change in category is made here from XPFORMULA to OPENXPPROP or CLOSEDXPPROP,
and from CLAUSE to SENTENCE.

I assume that the following attribute value pairs exist:

\begin{description}
  \item[mood] (declarative, yesnointerrogative, WHinterrrogative, Imperative,
       latenmood, Relative, anterelative, WHmod , $\langle$exclamative, narrative,
       optative$\rangle$ )
  \item[type] (mainclause, subordinateclause)
  \item[modus] (finite, omteinf, teinf, inf, imperative, participle, 
$\langle$subjunctive$\rangle$ )
\end{description}

Maybe the value {\em participle} must be replaced by two values: {\em prespart}
and {\em pastpart}. 

Values between $\langle\rangle$ are values that might be possible, but will not be 
used in Rosetta3 (in Dutch)

 Comments:
\begin{itemize}
  \item In fact already new attributes and their values have been defined 
already in domain  T. I have left this description as it was, although it must 
be changed somewhat.
  \item Anterelative: cf. Rosetta2
  \item WHmod : sentential modifiers of e.g. comparatives, {\em te},
tough-construction etc. that have a covert wh-phrase (sometimes, overt)at the
beginning, e.g.
\begin{enumerate}
  \item[] Piet kocht leukere dingen dan {\em  (WH) Peter verkocht} 
  \item[] Piet rekende op leukere cadeautjes dan {\em  waar Marietje aan dacht}
\end{enumerate}
  \item omteinf: all argument infinitival sentences that can begin with {\em om}
  \item teinf  : all infinitival sentences that have {\em te}, but cannot have
{\em om} (e.g. Hij heeft toegezegd (*om) te komen) 
  \item inf    : all infinitival sentences that do not have {\em te}
  \item The verb also has an attribute {\em modus}. Its values are set by the 
clauseformation rules.
  \item The values the attribute {\em modus} of the verb has can have when it enters this
rule class are: (finite, infinitive, imperative, participle).
  \item Notice that both the clause/sentence and the verb have the attribute modus. The
reason to have this attribute in both these categories is that their values can
be different, e.g. SENTENCE.modus=imperative, verb.modus=subjunctive in Spanish;
Sentence.modus=teinf, verb.modus=inf in Dutch. The attribute modus of clause
and the verb have been assigned a value already in the clauseformationrules.
The mooddetermination rules can change the value of modus in the SEntencerecord.
(e.g. from inf to teinf). Making the choice between (teinf, inf and omteinf)
here implies that a number of paraphrases are carried along only a very
short while (much shorter, than when this choice would be made in the 
clauseformation  rules).
\end{itemize}

\subsection{ The combinations (generative)}

 For every existing combination a condition(C)
is given that the inputmodel must satisfy. Furthermore, an action(A) is
specified.
 
All actions are accompanied by assignments to the attributes mood and
modus. The subdivision a. versus b. corresponds to sentence.type:= mainclause
and sentence.type:=subordinate resp. 
Sentences with a leftdislocrel/..  must still be taken into consideration.

                          will have to move further up.)
	 \end{enumerate}  %local 1

 The following preppatterns must be distinguished:
\begin{enumerate}
  \item obj is a NP      : in {\em de tuin}; 
   \item obj is a PREPP(PROP): voor {\em in de tuin}; van {\em voor de oorlog}; tot {\em in 1968};
   \item obj is a SENT(fin)  : zonder {\em dat hij het merkte}
   \item obj is a SENT(teinf): zonder {\em het te merken}
   \item obj is a ADVP(PROP) : tot {\em morgen}; vanaf {\em vandaag}; 
   \item obj is a R-word     : tot {\em hier}; vanaf {\em daar}
   \item arg-s are NP/PP (PP)PP: van Amsterdam/hier via Utrecht naar Groningen
\end{enumerate}

 For the case of PREPP-s and ADVP-s which can be imported only as PREPPPROP-s
and as ADVPPROP-s under the current assumptions, a decision must be made as to
what to do with their subject variables. One suggestion might be that the 
subject variable must bear the same index as the subject variable of the
PREPPPROP being formed, and that this variable disappears upon substitution of
the phrase, e.g.:
\begin{verbatim}

PREPPROP1 [ subj/VAR1 pred/PREPP [ head/PREP(*voor*) obj/VAR2 ] +
PREPPROP2 [ subj/VAR1 pred/PREPP [ head/PREP (*in*) obj/NP(*de tuin*) ]

-->
PREPPROP1 [ subj/VAR1 pred/PREPP 
                          [ head/PREP(*voor*),
                            obj/PREPP 
                                [ head/PREP (*in*),
                                  obj/NP(*de tuin*) 
                                ]
                          ]
 
\end{verbatim}

Semantically, however, I do not understand what this means

Depending on the preposition, the subject variable can be a sentence
(e.g. locative and temporal prepositions), or it must be interpreted
as some constituent in the sentence: e.g. in Ik heb {\em dit} gekocht voor
in de tuin, then {\em dit} seems to function semantically as the 'subject'
of [ voor in de tuin ]. (Is this really so?)

The preppatternrule must also take care of the fact that there are
{\em pre}positions, {\em post}positions and {\em circum}positions
('splitpreps') in Dutch, although it is also possible to take care of this
directly in the startpreppproprules. 

Splitpreps form PREPPROP-s of the following form:
\begin{verbatim}

                       PREPPPROP

                VAR1          PREPP1

                       PREPP2        PREP2


                     PREP1  VAR2  

\end{verbatim}

i.e. the first preposition and the following NP form a constituent. This
can be seen in such sentences as, where the two elements are shifted together:

\begin{enumerate}
   \item[]  {\em Naar wie} is hij {\em toe} gegaan
   \item[]  {\em Bij wie} is hij {\em vandaan} gegaan
   \item[]  {\em Onder welke brug} is hij {\em door} gefietst
\end{enumerate}

 The whole PREPPROP cannot be shifted (* {\em naar wie toe} is hij gegaan; 
* {\em onder welke brug door} is hij gefietst etc.), unless the relevant PP
is (according to the ANS) idiomatic, e.g. {\em Op het kantje af} heeft hij z'n 
examen gehaald ( I suspect that idiomaticity is not relevant here, but
'being an argument of the V' is).
 The last  prep in these cases starts to form a unit with the verb
(obligatorily so) cf. e.g. 

\begin{enumerate}
   \item[]  Ik weet niet naar wie hij toe is gefietst/ is toe gefietst
\end{enumerate}

This is more general for postpositions , cf{\em  ...dat Hij de garage in is
gereden/ is in gereden}, but not in the case of circumpositions:
{\em *...waar hij naar is toe gefietst, *...waar hij is naar toe gefietst }. 

Temporal postpositions ({\em terug, door, geleden}) can never be separated from
their objects ( cf. {\em een eeuw terug heeft hij dit gedaan} vs. {\em  *een
eeuw heeft hij dit terug gedaan} ) (cf. {\em  twee dagen geleden; de hele dag
door}) 

Certain prepositions are in fact circumpositions in disguise: e.g. {\em naar}
in its directional meaning is in fact {\em naar..toe}, where {\em toe} can be
left out if there is an NP following {\em  naar}, cf. {\em Waar ging hij naar
*(toe)? }.
Similarly perhaps {\em van} = {\em van...vandaan}, cf. {\em  Waar ging hij van
*((van)daan)}, and perhaps {\em door = door...heen}. 

The postposition {\em heen} only allows +R objects (in cases such as {\em door
het huis heen} there is the circumposition {\em door..heen }

I propose that a special type of PREPPPROP can be formed out of the words
{\em  er, hier, daar} etc. I propose to give these words the category RADV, and
to have a rule that forms a PREPPPROP out of a VAR and an RADV: PREPPPROP[
subj/VAR , pred/PREPP [head/RADV] ] 

Concerning the cases (c) and (d), I assume that these will be formed in the
CLAUSEtoSENTENCE subgrammar (mooddeterminationrules)

I propose to divide prepositions into several subtypes, and to compute
the subtype of the PREPP out of this information and information of the 
arguments of the PREPP
Some such subtypes of PREP-s might be:
\begin{description}
   \item[locative] binnen, in, door, onder , rondom etc.
   \item[directional] naar..toe, ..in, ..uit, heen, etc.
   \item[temporal] gedurende, tijdens, in, binnen etc.
   \item[according] ingevolge, krachtens, blijkens, overeenkomstig, ondanks,
etc.
   \item[topic] aangaande, betreffende, inzake, over, omtrent, etc..
   \item[excepting] behoudens, behalve, naast
\end{description}

{\em The values {\bf according } and {\bf excepting } are not in Domain T yet,
but they should be}.

Many prepositions (in particular the most frequent ones) are of several
subtypes (e.g. {\em in} is locational, directional (if postpositional), temporal
and perhaps even more. In these cases information about a typing of the
objects of the prep can help in disambiguating (e.g. cf. {\em in de tuin},
vs. {\em in een uur} vs {\em in deze zaak} etc.)

 A special way of using prepositional-like phrases is illustrated by
such cases as: {\em tegen de drie} huizen; {\em rond de vier}; {\em om en nabij de honderd}
boeken. I suggest that such cases are dealt with by a special subgrammar
that deals with approximative expressions, or that these constructions are
dealt with in the part of the grammar that deals with Numerals. A construction
such as {\em drie \'{a} vier} boeken  should also be accounted for there.

For a more complete list of the members of these subtypes, see file
[.iso]preps.dat

\section{ RC:Voice rules}
\begin{description}
  \item[HEADCATS] VERBPPROP
  \item[IMPORTCATS]--
  \item[EXPORTCATS] VERBPPROP
\end{description}

This rule class contains rules that turn the subjrel/VAR  into the
objrel/VAR of a byobjrel/PREPP, if the head verb allows passivization.
Variables that are going to be substituted for by EMPTY's are also subject
to this rule. The EMPTY is substituted, and deleted, or spelled out in some way
later on. The voice-attribute of the verbpprop can be assigned the value
{\em passive}, if one of these rules has been applied, in which case normal
passives are derived. However, the voice attribute can become active as well.
In this case we create structures for clauses like [ de boeken(door Piet)
verkopen ], which can appear as complements of the verb {\em laten} (e.g. {\em
Hij liet de boeken (door Piet) verkopen}). It is called: {\em DoorActive}. 

This is probably also the place to form [ aan x y lezen ] from [ x y lezen ]
for cases such as {\em Ik liet het boek aan Karel lezen/ zien} etc. (voice:
AanActive, only for a restricted set of verbs, only in combination
with {\em laten} 

 There is also a rule that simply makes the voice active, and further
changes nothing. The rule class is obligatory. This differs from 
earlier versions.

We might also make so-called {\em te-}passives as e.g. in {\em dat is te doen}
, {\em die zijn te verkopen } etc. In that case it makes sense to distinguish
between {\em kunnen-te } ({\em Dat is te maken}) and {\em moeten-te } (e.g.
{\em [de] te maken [opdrachten] }). We should also make passives then, where
overt modal verbs ({\em kunnen, moeten }) are introduced. These 'voices' 
are also necessary if we want to translate middle verbs from English.

The rule class RC:Voicerules must be applied before 
TC:ControlRules from the subgrammar VERBPPROPtoCLAUSE. It must be
applied before the TC:Zich-spelling as well.  

There are rules to form:
\begin{enumerate}
  \item Active Voice
  \item Passive Voice (impersonal and personal)
  \item DoorActive Voice 
  \item AanActive Voice 
\end{enumerate}

The attribute VERBPPROPREC.VOICE gets the appropriate value. The rule(s)
are applicable only if the attribute VERBREC.POSSVOICES contains the 
appropriate values.

In the XPPROPformation subgrammars (x=/=VERB) a default rule, making active
voice, is applied.

\section{ TC:Zichspelling}

\begin{description} 
  \item[HEADCATS] VERBPPROP, ADJPPROP, ?NPPROP, ?PREPPROP 
  \item[IMPORTCATS]--
  \item[EXPORTCATS] VERBPPROP, ADJPPROP, ?NPPROP, ?PREPPROP 
\end{description}

 This transformation class contains a rule that spells out the reflexive
pronoun ({\em  me, je, zich} etc.) if the verbpprop contains a reflexive verb.
By 
ordering this rule here, i.e. after RC:Voice Rules  and before
TC:ObjOKrules, it follows that reflexive verbs cannot be passivized: There is
no antecedent for the reflexive pronoun at this point in the derivation.

 The transformation class should contain a rule for reciprocal spelling
as well, to handle such cases as {\em Zij kwamen elkaar tegen} vs. {\em They
met} 

 This is probably also the best place to relate so-called bound variables
in idioms to their antecedents. If this is done, then it follows for these
as well, that passivization is not possible, cf.  
\begin{enumerate}
   \item[] Hij verloor z'n geduld  - Hij waste z'n handen in onschuld
   \item[] *Z'n geduld werd verloren-*Z'n handen werden in onschuld gewassen
   \item[] John lost his temper - *His temper was lost by John
\end{enumerate}
 cf. also idioms containing {\em zich}
\begin{enumerate}
   \item[] Hij verplichtte iemand aan zich vs. *iemand werd aan zich verplicht
   \item[] Hij maakte zich uit de voeten vs.* Er werd zich uit de voeten gemaakt
\end{enumerate}


So the members are: zich-spelling, elkaar-spelling, and rules for bound 
anaphors in idioms.

The antecedent of the reflexive will be: 
\begin{itemize}
  \item the argument as indicated by the attribute {\em  antecedent}, otherwise
  \item object, if there is one, otherwise
  \item subject, if there is one, otherwise
  \item there is no antecedent: rule blocks
\end{itemize}


Care should be taken of such verbs as {\em zich voordoen} as in {\em er deed
zich een nieuwe gelegenheid voor om ...  }
