\section{Introduction}
If one contemplates the market prospects of Rosetta, a few major problems keep 
coming back. For example, as a rule, free people will write ill-formed 
sentences.
Major additional stumbling blocks are the number of questions to be asked 
during interaction, and the inefficiency of the system.
It is my contention that we can not solve these problems.
Section~\ref{Cr} contains a few statements of belief that illustrate why I
think so. The market research performed by M und W was based on a product
we can not produce, it seems.

If one can not solve a problem, one may try to get around it. 
I write this note to convince myself that there {\em is} a way around, a
very natural way, and that a Rosetta translation system is viable.
If this (or some similar) trial will not be convincing, this means that our 
enterprise will be an academic one forever (which need not be long, at least
not for me). I hope this note, written in brain storm style, will provoke some
creative discussions.

In sections~\ref{Bu}, I argue that the strong and weak 
points of Rosetta make it the natural system for buiz letters, a kind of 
intermediate between standard letters and free letters. RosBuiz might be a 
nice name for a system intended to produce and translate such letters.
\section{A priori Credo}\label{Cr}
People are not Turing machines, and
\begin{itemize}
\item A Turing machine can not understand the world. 
\item The problem of ill-formed sentences exists because communication between 
      people degrades only gradually if sentences are incorrect, as people 
      {\em understand}. 
\item Ill-formedness can not be covered by a rule system.
\item One of Rosetta's strongest points is the refusal to do A.I. type of 
      semantics.
      However, the burden on interaction with the user is considerable, and it 
      will not be possible for us to translate a letter after an acceptable
      (to the average user) number of questions.
\item It is important that the user knows (explicitly and/or intuitively) about 
      the limitations of a RosBuiz system. The system should not be presented
      as being intelligent.
\end{itemize}
\section{Buiz letters}\label{Bu}
In the office, standard letters are used. Some time ago we have 
contemplated applying Rosetta to standard letters with free slots.
From the Rosetta point of view, anything can be a slot, but from the standard
letter point of view the number of slot types is limited. This means that
translating with Rosetta would be overkill, and I am pretty sure that, if
there is a market, someone will come up with a superior system tailored to 
standard letters.
However, the average buisiness (buiz) letters that need translation in an office
do not seem to be standard letters. Although letters for ordering something,
or acknowledgeing something, or letters accompanying a bill, etc., will have
much in common, standardizing them is something else. But if they have
much in common, their translations have much in common. 

Personally, I hate empty files. If I attempt writing a letter (or a program) I 
take another one, and modify it. Obviously, this is more true for hotel 
reservations than for letters to girl friends. But would you have your letter 
to your girl friend translated by a machine?

Whenever a letter has been disambiguated by consulting the user, the interlingua
expression corresponding to the letter is a very precious thing in terms of
user-effort. The interlingua version should therefore be stored with the letter.
If the letter is subsequently edited to produce a new letters, only the edited 
sentences need to be analyzed. Clearly, the most likely interpretation of the 
rest is the one that was chosen before.
Compared to a complete translation of the letter, this results in
a largely improved performance, with respect to both efficiency and 
the interaction burden, as the user is only asked about the changes he made. Of 
course, this is only true if not too many changes are made.
Also, chances are that the number of syntactic errors is small too,
at least compared to a letter produced from scratch. 
I suspect that most letters that are produced in the office world are in fact
small variations of other ones. 

I think that, assuming anaphoric references between sentences were solved
in the old letter, if an old reference is edited away, in most cases a new 
reference can be inferred without re-analyzing the anaphoric sentence.
The user is aware of the fact that the system interprets the current letter
with the previous one as its reference. He may click to inspect the current
interpretation of any word and change it if necessary. Thus, rather than 
writing a letter, the user is editing the interlingua on the basis of its 
representation in his language.

But how do I, a novel user of RosBuiz, start? My first letter can obviously
not be a copy of my previous one. Having to go through a laborious interaction
with the RosBuiz system about the letter I naively thought to be absolutely
clear and error-free is frustrating and will probably prevent me from using
the system ever. Especially so, because I am not too thrilled by computers
and never did any serious thinking about language. The error messages of
the RosBuiz beast are spaghetti to me. Could you not give me some examples?

A large data-base, with a good user interface, of example letters (including
standard letters) and example sentences will get the user going. As the
system gets used, letters produced in the user's office are added to the
data-base. Also, if letters are received that were translated by the
same release of RosBuiz, these can be added as well if the interlingua is 
transmitted along with the letter.
I think the average user will want to consult such a data-base if he writes
a letter unlike any previous one, even if it does not need any translation,
as he has always been stealing 'good' sentences from other people.
RosBuiz provides him with a good excuse to do so.

The user may install the most important prototypical letters as icons on his
screen. I think that the average user needs only a few prototypical letters,
in practice.

Letters are smaller than programs. Why not give the legal(!) user a limited
right to electronically consult RosBuiz B.V. if he really gets stuck with his 
letter. This feed-back would be useful to improve the system.

With each new release of the system, the user database can and should be 
updated automatically.

RosBuiz has a huge dictionary that is user-extendable. Note that to a large 
degree, the need for a domain-specific dictionary is replaced by the constancy
of interpretation of non-edited parts of prototypical letters.

Computers handling language are intriguing to people, and arouse interest in
human language itself. Therefore, RosBuiz has manuals at various levels of 
linguistic sophistication. I hope that a kind of RosBuiz cultus will arise, 
including the emergence of gurus.

RosBuiz signs each translation with 'translated by Rosetta', so that the 
rumor spreads automatically. 
\section{Conclusion}\label{Sc}
I am pretty sure that we could make a RosBuiz system in a few years time.
It will not enable the user to have any letter translated, but it will make 
possible a satisfactory communication with (buisiness) relations abroad, even if
the system has some linguistic gaps. Of course, the sequence of new releases
that improve the quality of this communication, is endless. 

Even if our ultimate goal would continue to be to make a translator for free
(E-mail) letters, a safe scenario would be to work on the Rosbuiz system
for the time being, and find out wether its quality warrants striving for
a general translator.
