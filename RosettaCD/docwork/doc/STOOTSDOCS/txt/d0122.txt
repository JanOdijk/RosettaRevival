$CH 3, Explanation of the GLUE-rules.
$ The GLUE-rules for Spanish are divided into the following types:$
$ TYPE MGLUE2: Middle GLUE-rules that divide a word into two segments.$
               e.g. de  + el :: del ;
                    con + mi¨' :: conmigo ;
               and Irregular Imperativo-forms + one clitic:
                    pon + (A) :: pon(A) ;
               (A) being a set of clitics.

$ TYPE MGLUE3: Middle GLUE-rules that divide a word into three segments.$
               e.g. pon + (B) + (C) :: po¨'n(B)(C) ;
               (B) and (C) again being sets of clitics.

$ TYPE MGLUE4: Middle GLUE-rules that divide a word into four segments.$
               e.g. pon + (D) + (E) + (F) :: po¨'n(D)(E)(F) ;
               (D), (E) and (F) being sets of clitics.

$ A similar three-partition is made for the Right GLUE-rules (RGLUE).
These rules are preceded by a *. To give an example, these tables contain
rules for composite forms of e.g. poner:$
               *po¨'n + (B) + (C) :: *po¨'n(B)(C) ;
$ (Compare this rule with the example under MGLUE3; the composite form has
already an accent on the stem).$

$ The Segmentation- and Lextree-rules will be specified in Chapter 4 (for
verbs) and 5 (for other categories).$

$CH 4, Introduction.
$ In this chapter an overview will be given of the preparatory studies which
led to the construction of the `Segmentation-rules` (i.e. suffix- and
freefixtable) and `Lextree-rules` for the Spanish verbs.$
$ In Chapter 4.1 a first classification of Spanish verbs will be given, 
independent of the Rosetta framework. In Chapter 4.2 the verbs will be 
organized in the light of their treatment in Rosetta. Finally in Chapter 4.3 a 
global summary of the rules will be given.$

$CH 4.1.1, Verbal characteristics.
$ First I made a provisional list of types of characteristics a Spanish
verb can have, providing each characteristic with a number:$


1)       o / ue   (mover)
2)       e / ie   (perder)
3)       e / i    (pedir)
3')      irregular subj.pasado, derived from irregular prete¨'rito

5)       n / ng   (poner)
6)       u / ue   (jugar)
7)       i / ie   (adquirir)
8)       l / lg   (salir)
9)       s / sg   (asir)
10)      o / u    (podrir)
10')     o / u only in gerundio  (poder)
11)      g / gu   (erguir)        (a general fonetic rule!!!)
12)       -ig-    (caer)
13)      "strong" prete¨'rito (decir)
14)      futuro:  poder  -->  podre¨'.
15)      futuro:  tener  -->  tendre¨'.
16)      futuro:  querer -->  querre¨'.
16')     "shortened" futuro: dire¨' (<decir) / hare¨' (<hacer).

21)      y / i  (y in presente. oi¨'r)
21')     y / i  (no y in presente. caer)
28)      i¨'  (ampliar --> ampli¨'o; efectuar -->  efectu¨'o)
29)      no accent (evacuar / apaciguar / abreviar)

30)      reunir -->  reu¨'no.

$CH 4.3, Rules for Spanish verbs in Rosetta.
$ After these preliminary study rules could be made for the Spanish verbs.
These rules consist of `Segmentation-rules` (freefix- and suffixtable) and 
`Lextree-rules`.
$ The rules will be given in Appendix 2 and 3.$
$ Here I will limit myself to briefly describing what types of rules
there are.$

$ For the verbs only non-recursive tables have been used.$
$ Furthermore it should be noted 
that inclusion of a certain (group of) key(s) in either the suffix- or
in freefixtable, isn't always well-motivated. E.g. an irregular 
Participle can be considered as having an irregular ending (and therefore
be included in the suffixtable) or as having a completely irregular form
(and therefore be a candidate for the freefixtable). A motivation for including
a key in one or the other table can be that both tables
are non-recursive (at least for the verbs), so each rule can only apply ones. 
If a verb has both an irregular stem and an irregular ending, the two relevant 
keys have to be included in different tables, so that both rules can apply.$

   --> Imperativo 2nd ps.sg.: SFKirrImperat2sg

              --> all regular verb endings. ( * + SFKo :: *o ; )

$CH 4.3.3, Lextree-rules.
$ In morphology an extra category ALOVERB has been introduced, with the
same values as VERB, in order to make it possible to use the rules for
regular V-endings for both "`stem` + regular ending" and 
"`allomorph` + regular ending".$
$ The following scheme can be made: $

(1)     VERB      ===>    ALOVERB + SFK
         |                   |
      SUBVERB             SUBVERB
         |                   |
       BVERB               BVERB

                                      |
(2)    ALOVERB    ===>    SUBVERB     |(3)   ALOVERB   ===>  SUBVERB  +  SFK
          |                  |        |         |               |
       SUBVERB             BVERB      |      SUBVERB          BVERB
          |                           |         |

$CH 5.1.1, Suffixtables.
$ For the non-verbal flexion there are two suffixtables, a recursive and a 
non-recursive one. For the 
nouns, adjectives and pronouns they contain the following Segmentation-rules.$

The `SUFFIX-table` (non-recursive):
- Rules for the irregular comparative forms, with SFKirrCompar, e.g.
             grande  +  SFKirrCompar  ::  mayor ;
- Rules for the irregular plural of interrogative pronouns, e.g.
             quie¨'n   +  SFKirrPL      ::  quie¨'nes ;
- Rules for the different number and gender of demonstrative adjectives and
  pronouns, e.g.
             ese     +  SFKmascDEMADJpl ::  esos ;

The `RSUFFIX-table` (recursive):
- Rules for pluralformation of both nouns and adjectives. 
              ( SFKregPL
                SFKirrPL
                SFKextranPL )
- Rules for ADJs which can have shortened forms, e.g.
     bueno  +  SFKapocope  ::  bueno ;
- Rules for the feminine forms of the ADJ, derived from the masc.form, with 
                                                                 SFKregFEM.
     
     NB. the i¨'simo-superlative and the mente-ADV are derivation, 
         rather than inflection.
