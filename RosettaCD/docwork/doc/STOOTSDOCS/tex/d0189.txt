\section {Problem I}
A few months ago Jan O. observed that Dutch words which contained
particles {\em and} derivational prefixes could not be analysed using the 
Rosetta3 morphology formalism. 
The morphological segmentation provides for two ways to separate affixes 
from their stems. The first way, by means of GLUE-rules, is meant for affixes 
which are considered to be words themselves, like the clitics "me" and "lo" 
in "damelo" or "not" in "cannot". The second way, by means of PREFIX and
SUFFIX-rules is meant for derivational and inflectional affixes. In the 
formalism the ordering between application of GLUE-rules and AFFIX-rules
is fixed, that is, first GLUE-rules are applied and then AFFIX-rules.

At the moment particles are separated in the Dutch morphology by means of
GLUE-rules. The reason for this is that they can also appear as seperate 
words in a sentence, e.g. "oplossen", "lost....op". However, this causes
problems if derivational or inflectional affixes can occur {\em before} the 
particle, because this would imply a rule ordering of the form 
AFFIX-rules $\rightarrow$ GLUE-rules. This phenomenon occurs in some Dutch
adjectives which are derived from verbs, like "onoplosbaar" and "onafleidbaar".
In the word "onopgelost" the situation is even more difficult because of the
affix "ge".
\section {Solution}
The most obvious solution of the problem is to treat Dutch particles in the
AFFIX-rules in stead of in the GLUE-rules. This measure enables the
segmentation to deal with an arbitrary ordering of particles and derivational
or inflectional affixes. One of the major consequences of this measure is
that the morphological analysis of the word "oplossen" yields just one S-tree
of the form:\\
\begin{verbatim}

                VERB
                
               SUBVERB

           PART      SUBVERB
           "op"      
                      BVERB
                     "lossen"

\end{verbatim}
\newpage
in stead of the S-tree sequence:\\
\begin{verbatim}

          PART ,  GLUE ,  VERB
          "op"            
                         SUBVERB
                          
                          BVERB
                          "lossen"

\end{verbatim}
In fact this measure causes a change of the interface between the
morphology and the M-grammar and it therefore should be looked at very 
conscientiously. Another consequence is that 
some extra Dutch lextree-rules have to be written for particle treatment.
These rules will be of the following form :\\
\begin{verbatim}

    PART           SUBVERB                   SUBVERB
             +                   ---->
                    BVERB                  PART     SUBVERB
     
                                                     BVERB
\end{verbatim}
or
\begin{verbatim}

    PART             VERB                        VERB
             +                   ---->
                    SUBVERB                     SUBVERB
     
                     BVERB                    PART     SUBVERB

                                                        BVERB
\end{verbatim}
\section {Problem II}
The result of application of the analytical morphological segmentation 
component to one word is a row of affix keys and basic keys. The value of the
basic keys is specified in MDICT, whereas the affix keys are specified in the
segmentation rules. Next, in the ALEX 
phase, for each key the corresponding S-tree is created. Basic keys correspond
to one entry in B-LEX, containing the same basic key. Affix keys correspond to 
one entry in AFFIX-LEX and because AFFIX-LEX is language specific just as 
B-LEX, it must be specified for each language. A lot of entries in the 
AFFIX lexicon are trivial. They just specify that with an affix key K an
S-tree SFCAT[key:K] or PFCAT[key:K] corresponds. Some of the entries in
the lexicon, however, are not trivial because they make a connection between
affix keys and B-LEX entries. E.g. if we treat particles in the way suggested in
the previous sections there will be one affix key for each
particle. Furthermore, there will also be a corresponding B-LEX entry because
particles can also occur in isolation. Other examples are some 
derivational affixes that are treated as basic expressions by the M-grammar.

The problem is that specifying the BLEX information at two places, both in 
B-LEX and in AFFIX-LEX, would cause redundancy,
which makes the lexicon more sensitive for inconsistencies.

\section {Solution}
In DOMAIN-T we distinguish two sets of affix keys: M-AFFIXES (for 
morfological affix keys, e.g. inflectional affixes) and S-AFFIXES ( for 
affixes that also occur in the syntaxis ). Furthermore, the keys corresponding
to these affixes will be treated differently in the ALEX phase. 

\begin{itemize}
  \item M-AFFIXES will be looked up in AFFIX-LEX. In order to make this
lexicon language independent, the syntactic 
categories SFCAT, PFCAT and GLUE will be part of the formalism, which means
that they don't have to be specified in DOMAIN-T. The AFFIX-LEX function that 
makes an S-tree for an M-AFFIX key is defined as follows:
\begin{verbatim}
AFFIX-LEX(k) =
              { t | Kind(k) = suffix and t = SFCAT[key:k] } +
              { t | Kind(k) = glue   and t = GLUE[]}        +
              { t | Kind(k) = prefix and t = PFCAT[key:k] }
\end{verbatim}
  \item For S-AFFIXES there are (at least) two alternative solutions. 

The first
solution requires a lexicon in which for each S-AFFIX a B-LEX key is specified.
Next, B-LEX is searched for the corresponding B-LEX entry. 

In the second
solution the value of S-AFFIX key is already the same as the value of the
B-LEX key. This can be achieved by specifying it in DOMAIN-T during the
definition of the S-AFFIX keys. This implies that the translation of S-AFFIXES
into S-trees can be performed in exactly the same way as the translation
of ordinary basic keys.

A disadvantage of the first solution is that it requires an extra lexicon,
for which we have to invent a notation and for which we have to write some
kind of compiler.
\end{itemize}
