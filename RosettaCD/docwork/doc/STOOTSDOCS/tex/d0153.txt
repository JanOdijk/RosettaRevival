\section{ TC: Aux-to-Comp}

\begin{description}
   \item[HEADCATS] SENTENCE
   \item[IMPORTCATS]--
   \item[EXPORTCATS]SENTENCE
\end{description}

 This transformation class turns the verb functioning as a auxrel/..
into a verb functioning as a conjrel/.. in main clauses. It does so
if there is a shiftrel/.. (wh) and there is a subjrel/.. 
or if the sentence has mood {\em yesnointerrogative}. The transformation class 
lifts the auxrel/.. over the subjrel/.. to the left:

\begin{verbatim}

   [ shiftrel/..,                subjrel/.., auxrel/VERB1, .... ] -->
   [ shiftrel/.., conjrel/VERB1, subjrel/..,               .... ]
\end{verbatim}

This accounts for such cases as : {\em do} you leave now? ; what {\em did} you
see? etc.

\section{ TC:Bound anaphorcheck}

\begin{description}
   \item[HEADCATS]VERBPPROP
   \item[IMPORTCATS]--
   \item[EXPORTCATS]VERBPPROP
\end{description}

 This transformation class supplies the correct values to bound variables in
idioms to their antecedents. By ordering this operation here, it follows, that
passivization is not possible for such constructions, if the subject is the
antecedent of the bound anaphor: At this point in the derivation there is
no subject if a passive sentence is being formed, cf. 
\begin{itemize}
  \item  John lost his temper - *His temper was lost by John
\end{itemize}

 If English has reflexive verbs, or reciprocal verbs, then the reflexive
element or the reciprocal element is spelled out in this transformation class,
cf. in Dutch: TC:Zichspelling, and in Spanish: TC:Se-spelling

\section{ RC:Clause-formation}

\begin{description}
   \item[HEADCATS] VERBPPROP, ADJPPROP, ADVPPROP, NPPROP, PREPPPPROP
   \item[IMPORTCATS] To be specified by Lisette Appelo
   \item[EXPORTCATS] CLAUSE, VERBPFORMULA, ADJPFORMULA, ADVPFORMULA, NPFORMULA,
PREPPPFORMULA 
\end{description}

 These rules turn the VERBPPROP into a CLAUSE, introduce time and aspect
and spell these out by means of changing attribute values and introducing 
auxiliary verbs

In English aspect is spelled out by the auxiliaries {\em have} +past partic.,
(for perfect); {\em be} + ing-form for progressive. In case the sentence is
passive, the passive auxiliary {\em Be} + past participle is introduced.

These auxiliaries are positioned in the following way:
\begin{enumerate}
  \item[] subj/VAR, haveauxrel/HAVE, ingberel/BE, passauxrel/BE, pred/VERBP
\end{enumerate}

(cf.  {\em I have been being tortured by them})

It should be taken care of that the verb following {\em have} is turned into
a past participle, the one following ingberel is turned into an ingform,
and that the verb following passauxrel is turned into a past participle.

In finite sentences one of the following two rules must apply:

If no haveauxrel, ingberel or passauxrel is present, insert auxrel/DO
between subjrel/.. and predrel/VERBP. Mark {\em do} as being of present or past
tense (do, did). This {\em do} will remain here until the transformation of
Do-Deletion, that makes the main verb in stead of {\em do} finite and
deletes {\em  do} if {\em do} is
adjacent to the main verb and not emphasized.

If either haveauxrel, or ingberel or passauxrel is present, the make the
leftmost of these bear the relation auxrel:

\begin{enumerate}
  \item[] CLAUSE\{\}[ sig1, rel1  /T, mu2, predrel/VERBPPROP]  $\rightarrow$
  \item[] CLAUSE\{\}[ sig1, auxrel/T, mu2, predrel/VERBPPROP]  
  \item[] COND rel1 in [haveauxrel, ingberel, passauxrel ]\\
          sig1 = eps or sig1 = subj/..
\end{enumerate}

and mark this verb for present or past 

 If no haveauxrel, ingberel or passauxrel is present, and the main verb in
VERBP is {\em have}, then this verb can be lifted out of VERBP to bear the
relation auxrel. In this way we can account for the possibility of
inversion without do-support of the main verb {\em have} (e.g. {\em Have I 
a book?})
(what about: {\em Have I to leave?}). This is typically a rule that one would like to
do in analysis only.

In infinitival sentences, either no auxrel appears (e.g. for 
{\em I saw [ him come]}), or the element {\em to} functions as such (e.g. 
I promised [ {\em to} go away]). This element {\em to} will be introduced 
later in the mooddeterminationrules.

If the predrel/.. is occupied by NP,ADJP, ADVP, or PREPP, then copular
sentences are created by introducing the auxiliary verb (copula) {\em be}. This
verbs's main function is to function as a carrier of tense and aspect. The
auxiliary {\em have} can be introduced as well (for perfect aspect), and perhaps
progressive {\em  be} can as well. The passive auxiliary cannot appear of course:

\begin{itemize}
  \item subjrel/VAR, haveauxrel/HAVE, ingberel/BE, copularel/BE, predrel/XP
\end{itemize}

 (e.g. {\em I have been ill; ?I have been being ill})

It appears as if the copula {\em be} and the passive auxiliary {\em be} 'occupy
the same position'. If this is correct, then we might make them bear the same
relation

Exactly as in the case where predrel is a VERBP, the leftmost of these
auxiliaries must start to bear the relation auxrel

In XPPROPtoXPFORMULA subgrammars a default rule applies that has the same
meaning as the rule that introduces the copular verb {\em be} in present tense
and as the rule that introduce the auxiliary {\em do} in present tense,
and as the rule making infinitives with dependent tense.

\section{ TC: Do-deletion}

\begin{description}
   \item[HEADCATS] SENTENCE
   \item[IMPORTCATS] --
   \item[EXPORTCATS]SENTENCE
\end{description}


 This transformation class copies the features person, number of the
auxiliary verb {\em do} onto the main verb, which is made finite, and it deletes
{\em do}. This should happen only in the configuration:
\begin{itemize}
  \item ...auxrel/do predrel/VERBP\{\} [ head/VERB ... ]...
\end{itemize}

 i.e. if {\em do} and the main verb are adjacent.

The transformation class is either applied obligatorily , or it is an optional T
+ Filter Transformation

\section{ RC:Makeadvsent}

\begin{description}
   \item[HEADCATS] PREP, CONJ
   \item[IMPORTCATS] SENTENCE
   \item[EXPORTCATS] ADVSENT, PREPP
\end{description}

This rule class makes adverbial sentences (ADVSENTs), or PREPP-s containing a
sentence that are going to function as adverbial sentences out of
PREP-s, CONJ-s and SENTENCES

There is no specific reason to make this a special subgrammar, but
it seems more transparent to do so. This is particularly the case since it
might be the case that PREP-s must be translated into CONJ-s

Another natural place where the relevant effects can be achieved is in the
Mooddetermination rules 

 The rules combine a.o.
\begin{itemize}
  \item prepositions such as {\em before}, {\em after}, {\em until}, {\em
except } with finite declarative sentences (where {\em that} must be deleted in
most cases ) to form PREPP-s; 
  \item conjunctions such as {\em while}, {\em if}, {\em once} with finite
declarative sentences to form finite ADVSENT-s 
  \item prepositions such as {\em without}, {\em after}, {\em before} etc. with
a declarative ingsentence (open-ing or acc-ing) to form PREPP-s. 
\end{itemize}

 Examples:
\begin{enumerate}
  \item[]  ..before he left ; .. except that there is no food
  \item[]  ..if we do that ; ..while he is preparing this
  \item[]  ..without looking around ; ..without him being present
\end{enumerate}

For the conjunction {\em as}, see doc. R0152: {\em zoals}.

\section{ RC: Mooddetermination}
\begin{description}
   \item[HEADCATS] CLAUSE, VERBPFORMULA, ADJPFORMULA, ADVPFORMULA, NPFORMULA,
PREPPPFORMULA 
   \item[IMPORTCATS]--
   \item[EXPORTCATS] SENTENCE, OPENVERBPPROP, CLOSEDVERBPPROP, OPENADJPPROP,
CLOSEDADJPPROP, OPENADVPPROP, CLOSEDADVPPROP, OPENNPPROP, CLOSEDNPPROP,
OPENPREPPPPROP, CLOSEDPREPPPPROP 
\end{description}

This rule class determines the mood and the type of the sentence, and
spells out conjunctions, if required.

I assume that the following attribute value pairs exist:
\begin{description}
  \item[mood] (declarative, yesnointerrogative, WHinterrrogative, Imperative,
       latenmood, Relative, anterelative, WHmod ,  $<$exclamative, narrative,
       optative$>$ )
  \item[type] (mainclause, subordinateclause)
  \item[modus] (finite, fortoinf, toinf, inf, imperative, pastparticiple,
open-ing , acc-ing, poss-ing, subjunctive ) 
\end{description}

 Values between $<>$ are values that might be possible, but will not be 
used in Rosetta3 (in English )

 Comments:
\begin{itemize}
  \item  Actually, the mooddeterminatiuon rules will apply differently, see
  e.g Domain T. The description given here must be adapted to that.
  \item Anterelative: cf. Rosetta2
  \item WHmod       : sentential modifiers of e.g. comparatives, {\em too} etc.
that have a covert wh-phrase (sometimes, overt) at the beginning. 
  \item fortoinf: all infinitival sentences that can begin with {\em for}
             e.g. {\em [I prefer] for him to come}
  \item toinf : all infinitival sentences that have {\em to}, but cannot have
{\em for}, e.g. {\em [I believe] him to be the worst president I've ever
seen} 
  \item inf    : all infinitival sentences that do not have {\em to}
   \item open-ing: sentences containing ing-form and a nonovert subject:
                {\em [I like] swimming};        
   \item acc-ing: Sentences containing an ing-form and a subject in accuastive
Case: {\em  [I saw] him coming} 
  \item poss-ing: Constituents containing an ing-form and a subject in
possessive Case: {\em His going home [surprised me]} 
The category of such constituents is probably NP, not Sent. Its 
internal structure might be : NP [ subjrel/NP , predrel/VERBP ]
\end{itemize}

 Adverbial sentences are formed by the ADVSENT subgrammar

\subsection{ The combinations}

For every existing combination a condition(C) is given
that the inputmodel must satisfy. Furthermore, an action(A) is specified.
All actions are accompanied by assignments to the attributes
mood, type, and modus, as indicated by the surrounding actions.

The verb also has an attribute {\em modus}. Its values are set by the 
clauseformation rules.

\section{ TC:Objectokrules}
\begin{description}
   \item[HEADCATS] CLAUSE, VERBPFORMULA, ADJPFORMULA, ?ADVPFORMULA, ?NPFORMULA
   \item[IMPORTCATS]--
   \item[EXPORTCATS] CLAUSE, VERBPFORMULA, ADJPFORMULA, ?ADVPFORMULA, ?NPFORMULA
\end{description}


 This TC contains a.o.
\begin{itemize}
  \item  T:Object/Indobj-promotion: turns an obj/ or an indobj/.. into a subj/
in the case of passives and ergatives if it is adjacent to the verb. 
 Examples:
      \begin{enumerate}
	\item[] was accused Peter by his friend $\rightarrow$\\
		Peter was accused by his friend
	\item[] was given John a book           $\rightarrow$\\
		John was given a book
      \end{enumerate}

  \item T:Prepobj-promotion: It turns the obj/.. in a prepobj/.. into  a subj/..
if the verb is passive and if the prepobj/.. is adjacent to the Verb
 Examples:
\begin{enumerate}
  \item[] was looked at Peter by Mary $\rightarrow$\\
          Peter was looked at by Mary
\end{enumerate}
 The adjacency requirement must be relaxed slightly in the case of idiomatic
(or perhaps semi-idiomatic) constructions, e.g.:
\begin{enumerate}
  \item[] has been taken {\em care} of the girl $\rightarrow$\\
          the girl has been taken care of 
  \item[] has been taken advantage of Bill  $\rightarrow$\\
          Bill has been taken advantage of
\end{enumerate}

 If we assign the following structure to such cases:
\begin{enumerate}
  \item[] VERB(*take*) NP(*care,advantage*) prepobj/PREPP
\end{enumerate}

and if we relax the adjacency requirement for prepobj/..-s by allowing an
NP-part(consisting only of a noun?) of an idiom to intervene between the verb
and the prepobj/.., then both well-formed passives of these idioms can be
derived: 
\begin{enumerate}
  \item[]  {\em he has been taken advantage of}\\
           is derived by T:Prepobjpromotion, and
  \item[] {\em Advantage has been taken of him}\\
          is derived by T:Objectpromotion
\end{enumerate}

 Notice that one can see whether some obj/.. is part of an idiom or not
because, if it is not, then it still is a VAR, otherwise it is a NP.
If, however, it is a semi idiom, then this cannot be seen.

  \item T:There-insertion: it puts NP[head/THERE] into the structure bearing
the relation subjrel, in case there is no subjrel, the verb is {\em be}, or one
of a small class containing {\em arrive}, {\em come} etc. (these with lowering
of the bonus, I suggest, and/or only analytically), and the obj/.. is
indefinite. The NP dominating THERE is assigned the number of the postverbal NP
.


  \item  T:NP-raising: it turns a subjrel/ from the deeper (by one level of
embedding) proposition into the subjrel/ of the higher VERBPPROP,e.g. in the
case of raising verbs, passives, copular verbs ({\em seem, turn out} etc.): 

\begin{enumerate}
  \item[] [ head/seem, compl/CLOSEDTOSENT [ he to be intelligent ] ]$\rightarrow$\\
  \item[]  [ subj/he head/seem, compl/CLOSEDTOSENT [ to be intelligent ] ]
\end{enumerate}

\end{itemize}

\section{ TC: ParticleHop}
\begin{description}
   \item[HEADCATS] CLAUSE, VERBPFORMULA
   \item[IMPORTCATS]--
   \item[EXPORTCATS] CLAUSE, VERBPFORMULA
\end{description}

 This transformation class optionally hops the particle over the adjacent NP
to the left: 

 The configuration V- NP - PART  is turned into V- PART -NP

 The NP may not contain a personal pronoun (cf. *He called up him)

 It is assumed that particles are put behind the first NP following
the verb, if such an NP is present, and directly following the verb if no
NP is present at the moment that they are spelled out in the particle
Insertion transformation).

\section{ TC: ParticleInsertion}
\begin{description}
   \item[HEADCATS] VERBPPROP
   \item[IMPORTCATS]--
   \item[EXPORTCATS] VERBPPROP
\end{description}

This transformation class spells out the particle, if there is one.
It puts the particle directly behind the verb, unless the verb is followed
by NP: In that case the particle is inserted behind this NP. Here we follow
Emonds (1985:252-256). 

Later in the derivation the transformation Particle Hop optionally hops the
particle over a preceding NP, if there is one, and if it is not a pronoun.

The members of this TC are:
\begin{enumerate}
  \item T:ParticleSpelling
  \item T:NoParticleSpelling
\end{enumerate}

Reference:
\begin{enumerate}
  \item[] Emonds, Joseph, E. : {\em  A unified theory of syntactic categories}.
          Dordrecht: Foris
\end{enumerate}

Infinitival sentences are subject to Extraposition, optionally, with the

Sentences cannot be introduced as an obj of a prepobj/..., unless they ar

Special rules must be added to account for the proper incorporation of
'syntactically modal' verbs.

To be more specific, we distinguish  semantic modals  and 
syntactic modals . Semantic modals are all verbs with a modal meaning.
Syntactic modals are verbs from a very specific list, the main members of
which are given below:
\begin{description}
  \item[SYNTMOD] = [ can, could, may, might, must, must, will, would, shall,
should ] 
  \item[SEMMOD]  = SYNTMOD + [ be able to, be allowed to, be obliged to,
want,etc. ] 
\end{description}

All verbs with a modal meaning either take only one argument (a sentential
complement) or two arguments (a subject and a sentential complement)

There are special members in the TC:PropOkrules for the case where a sentential
complement is an argument of a member from SYNTMOD.

Suppose the proposition insertion rules and clause formation rules have yielded
a structure of the form:
(This is an example structure, for convenience certain variables (e.g. for
adverbs) have been left out in this example sentence) 
\begin{verbatim}

CLAUSE [ subj/T1, pred/VERBP 
                       [ head/VERB1, 
                         compl/SENT [ subj/T2, 
                                      pred/VERBP[
                                           head/VERB2,
                                           mu1
                                                ]
                                    ]
                       ]

       ]
\end{verbatim}

where VERB1 in SYNTMODS, and T1=T2 (example: x1 can [ x1 buy x2 ] ), then
this structure is changed into: 

\begin{verbatim}

CLAUSE [ subj/T1, auxrel/VERB1, pred/ VERBP
                                    [ head/VERB2,
                                      mu1
                                    ]
       ]
\end{verbatim}

(subj/x1 aux/can pred/VERBP [ head/buy x2 ])


Since PropOKrules apply after Clause formation, the modals {\em can } replaces
{\em do}.

See also the note on modals in R3 by Jan Odijk, to appear.

\section{ RC:Voice rules}
\begin{description}
  \item[HEADCATS] VERBPPROP
  \item[IMPORTCATS]--
  \item[EXPORTCATS] VERBPPROP
\end{description}

This rule class contains rules that turn the subjrel/VAR  into the
objrel/VAR of a byobjrel/PREPP, if the head verb allows passivization.
Variables that are going to be substituted for by EMPTY's are also subject
to this rule. The EMPTY is substituted, and deleted, or spelled out in some way
later on. The voice-attribute of the verbpprop is assigned the appropriate
value.

 There is also a rule that simply makes the voice active, and further
changes nothing. The rule class is obligatory. This differs from 
earlier versions.

 The rule class RC:Voicerules must be applied before 
TC:ControlRules from the subgrammar VERBPPROPtoCLAUSE. It must be
applied before the TC:Boundanaphor check as well.  

We might also assume a {\em middle voice}-formation rule here. We might assume
that certain verbs are marked in such a way that they allow middle voice
(e.g. in the possvoices attribute). Such verbs have the attributes 
thetavp=vp120 and synvps=[DONP]. The middle voice rule turns the subject,
if it contains an EMPTYVAR, into a byobj. The EMPTY substitution rule
can be applied later on only if there is an adverb of a certain kind
(vpadv) present. Hence we can derive {\em Bureaucrats bribe easily } but
not {\em *bureaucrats bribe } or {\em *bureaucrats bribe easily by them }.
The time rules should be restricted accordingly.

From a translational point of view, this does imply that there is e.g.
a {\em te-passive } or modal passives in Dutch.


The attribute VERBPPROPREC.VOICE gets the appropriate value. The rule(s)
are applicable only if the attribute VERBREC.POSSVOICES contains the 
appropriate values.
