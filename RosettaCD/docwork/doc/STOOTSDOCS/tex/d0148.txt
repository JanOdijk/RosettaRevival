\section{Introduction}
This note has been started as an inventarization of the pronouns of the Van 
Dale, in order to determine whether new categories were necessary in Domain
T. The scope
of the note has extended slightly compared to this original aim.
The pronouns occurring in the VanDale are treated and assigned
a category. Only a specific class of pronouns that probably belong to
the category BDET or BQ (See doc. 0141,0146) have not been assigned a category
here. They are listed in the last section. 
A proposal is made to extend Domain T slightly. 


\section{ Pronoun categories in Domain T (doc. 0141,0146)}

In the docs. 0141, 0146 the following categories for pronouns
exist. In this section an exhaustive list of their members will be given.
{\em Exhaustive } means: to the extent that they occur in the Van Dale
with a category code starting with {\em 4 } as extracted by Harm Smit.
Roman Numbers behind a word refer to the subclassification made in the 
{\em  Van Dale}. 
\begin{description}
  \item[BIGPRO] abstract basic category. Its members are abstract elements:
	 \begin{itemize}
	   \item {\em ikbigprokey, jijbigprokey, hijbigprokey, zijsgbigprokey,
                 wijbigprokey, julliebigprokey, zijplbigprokey, jebigprokey,
                 zebigprokey}
	 \end{itemize}

  \item[BDEMADJ] Its members are: {\em die I,deze, dit, dat I, gene }
         They all have code 42 in Van Dale. There are other words with code
          42. 
         Van Dale does not distinguish between BDEMADJ and BDEMPRO, so that
         all of the words mentioned must be split up
         In contrast to a 
        suggestion made in doc 0141, {\em zulk } does not belong to this class.

  \item[BDEMPRO] Its members are {\em  dit , dat I} , {\em die }
            and {\em  deze}. See also BDEMADJ.
  \item[BINDEFPRO] Its members are: {\em allehens, alleman, alles, andermans,
elkeen, ieder, een ieder iedereen,  een iegelijk, iemand, iets, ietwat, men,
menigeen, niemand, niemendal, niets, niks, nop, noppes, wat    III, zowat }.
The following remarks can be made: 
\begin{enumerate}
  \item {\em allehens } only in the expression {\em allehens aan dek }
  \item {\em alleman } only in genitive and in {\em Jan en alleman }
  \item {\em anderman} occurs in the genitive only: {\em  andermans zaken}
  \item {\em ieder } as in {\em ieder van ons } and in {\em in ieders belang}.
      It can also be used adjectivally.
  \item {\em een iegelijk } In the Van Dale as {\em iegelijk }. Occurs also
        in the expression {\em elk en een iegelijk }. It can be used 
adjectivized as well, according to the Van Dale, so here a splitting up should
be done cf. {\em een iegelijk mens } 
  \item {\em ietwat } According to Van Dale it can mean {\em iets }. No 
examples given. In my opinion, it can be used only in such cases as 
{\em  ietwat te groot}, but not e.g. in {\em *Hij zag ietwat }. Therefore
I propose not to call it an INDEFPRO, but an BADV, or perhaps a BQ.  
  \item {\em men }. In contrast to what has been done in doc. 0134, I consider
        {\em men  } to be an INDEFPRO.
  \item {\em nop, noppes }. Only(?) in the expression {\em voor nop(pes) }.
I propose to remove them from the dictionary, and to add the fixed idioms
{\em  voor nop(pes)}
  \item {\em zowat } only in the expression {\em en zowat }
\end{enumerate}
    All members have code 48 in the Van Dale, but there are also other words
     with code 48. Genitive forms, such as {\em iemands, iedereens } etc. 
should be added. These are not Basic expressions, of course.
 
                                     
  \item[BPERSPRO] The members are: {\em  gijlieden, ze II, wij, je II, hij,
jullie I, zij (sg), zij(pl), het I, het II, gij, jij, ik, u, hunnie }. All have
code 42, except {\em het II },{\em  je II} and {\em ze II } which have code 48.
For {\em het I } and {\em het II }, see below. The cases {\em je II } and {\em
ze II} represent cases of {\em  je, ze} used in a general sense. The ANS also
has {\em hullie, jelui }.

  \item[PERSPRO] Some must be in the lexicon, but most are derived in the 
morphology. The members are: {\em ikke,
gijlieden, haar I, ze II, mezelf I, wij, ge, me, we , je II, jou, ons I, hij,
jullie I, ze I, zij (sg), zij (pl), het I, gij, hen, hunner, mij I, je I, jij,
ie, ik, hun I, u, hem, zullie.} 
Remarks: Van Dale distinguishes between {\em zij(sg) } and {\em zij(pl) }
only within an entry. The words {\em mezelf I } and {\em jezelf} are listed as
emphatic pronouns, but e.g. {\em hijzelf, hemzelf, jezelf } etc. lack. I
propose to treat these as pronoun + GLUE + {\em zelf}. {\em Hunner } is a
genitive form of {\em hun }. I propose to remove this word out of the
dictionary. 
  \item[POSSADJ] The members should not be in the R3 lexicon, since they
   are derived from BPERSPRO's in the morphology. Van Dale lists the following:
{\em    hun II, haar II, uw, jullie II, je III, mijn, zijn, ons II, jouw}
 The morphology can handle {\em onze, } and genitive forms ({\em mijner, jouwer
etc. }) as well. Furthermore, the morphology can handle the forms {\em  wiens,
wier} and {\em diens, dier} ( {\em  wiens, wier} are not in Van Dale, except
that {\em wiens} occurs in an example under the entry {\em wie}). 
They all have code 43. No other word has code 43.
   \item[POSSPRO] The members should not be in the R3 lexicon, since they
   are derived from BPERSPRO's in the morphology. Van Dale lists none of them.
They are mentioned under the corresponding POSSADJ entries. 
   \item[RECIPRO] The members are: {\em elkaar, mekaar, malkaar}. Genitive
forms exist as well. These must be listed in the dictionary as well:
{\em elkaars, mekaars, malkaars}. Their code is 47. No other word has code
47. {\em elkander, elkanders } are not in Van Dale.
   \item[REFLPRO] Van Dale gives only {\em mij II, zich, mezelf II,
zichzelf}.  They have code 46.
The following should be added: {\em  mijzelf, me, je(sg.), jezelf(sg.),
ons, onszelf,
je(pl.), jezelf(pl.), u, uzelf, jezelf}.
   \item[RELPRO] Its member are {\em wat I, hetgeen II, wie II, welk II, 
dat II, dewelke, hetwelk, die III}. They all have code 44. No other words have
this code. The following should be added: {\em hetgeen I} (code 42, 'aanwijzend
voornaamwoord' in Van Dale) The distinction Van Dale makes here should not be a 
distinction between relative and demonstrative pronouns, but between the use
of {\em  hetgeen} in a free relative ({\em Ik blijf bij hetgeen ik gezegd heb }
) versus its use as a relative pronoun with a sentence as antecedent:
({\em Hij ging naar huis, hetgeen ik betreur }).
   \item[BWHPRO] Its members are {\em wat II, wie I}. They both have code
45. Some other words have this code as well.
\end{description}

\section{New categories}

I propose to add the following category:
\begin{description}
  \item[RELANTPRO] Its members are:{\em diegene, degene, datgene, degenen,
diegenen }. These are
pronouns that must be combined with a relative clause. They are the ANTecedent
of a RELative clause. 
  Van Dale has the word {\em die II } as a 'bepalingaankondigend voornaamwoord'
(its code is 00!). It is unclear what is meant. I suggest to drop it. Maybe
the use of the pronoun {\em die } in such sentences as {\em Hij liet {\em die }
meisjes met {\em die } jongens dansen, die elkaar nog van vroeger kenden }. 
is intended. Such sentences will not be dealt with in R3.
\end{description}

The category RELANTPRO has a RELANTPROrecord, with the attributes: 
\begin{itemize}
  \item nform:nformtype:false
  \item KEY
\end{itemize}
 
The attribute {\em nform } is required to distinguish {\em degenen, diegenen } from
{\em  degene, diegene}.

\section{Changes in Domain T}

I propose to adopt the following further changes. These changes are independent
of the rest of the proposals in this document, so they can be decided on 
independently.
\begin{itemize}
  \item I propose to drop the attributes {\em definite, sexes, number, 
detnpmood} from BWHPROrecord, so that a KEYrecord is left over. These 
attributes are either superfluous ({\em  definite, detnpmood}) or required for
POSSADJ's and POSSPRO's only ({\em sexes, number}). The consequence is that
neither {\em wie } nor {\em wat } are ambiguous anymore w.r.t. number, sex.
There must be two rules now to form an NP out of {\em wie }. One rule forms
a singular NP headed by {\em wie } (Cf. e.g. {\em Wie heeft dat gedaan }), and
one forms a plural NP headed by {\em wie} (Cf. e.g. {\em Wie hebben dat gedaan
}). 
The word {\em wat} can head singular NP's only (cf. {\em wat irriteert hem?} 
vs. {\em *wat irriteren hem?}). In cases such as {\em Wat zijn de 
consequenties?} the plural form is caused by the NP behind the verb.
(cf. {\em wat zijn/is de consequentie/consequenties?}).
  \item In combination with the first change proposed, I propose that
the attributes {\em sexes, number } are added to the POSSADJrecord.
These are necessary to distinguish between {\em  wiens} and {\em wier },
and between {\em diens } and {\em dier }.
We might also consider the possibility to split POSSADJ up into POSSADJ and 
WHPOSSADJ, as in Rosetta2, because it would make the records more 'natural'.
   \item Again, in combination with the changes above, we can now remove the
attributes {\em sexes, number} from BDEMPROrecord. For {\em die, deze } there 
must now be two rules to form NP's, one for singular and one for plural NP's.
For {\em dit,dat} this is not necessary, since they always head singular NP's
(cf. {\em *dit bewijzen de stelling}). For cases such as {\em dit, dat zijn
de problemen }, it is the second NP that determines the number upon the verb
(Cf, {\em dit,dat is het probleem/*de problemen}. We must, however,
mysteriously enough, allow for such cases as : {\em wat zijn dat, dit?}, 
although, as we have just shown, neither {\em wat } nor {\em dit, dat} 
head plural NP's. Such sentences can be taken care of separately in the
Identificationalpropformation subgrammar, and/or in the subject-verb agreement
rule.
    To be more precise, we must also take care of cases where the {\em dit,dat}
NP 'runs away' from its original position, as in {\em dit lijken de problemen 
te zijn}. Here the distance between the finite verb and the postverbal NP 
becomes to big. Therefore it is necessary to copy the number feature of the
postverbal NP onto the NP containing {\em dit,dat,het}. the subject-verb 
agreement rule can then apply normally, referring to the subject NP. Thus,
typical structures are now:
\begin{enumerate}
  \item VERBPPROP[subj/NP\{number:sg\}[head/dit], head/VERB, idrel/NP\{number:
sg\}]
  \item VERBPPROP[subj/NP\{number:pl\}[head/dit], head/VERB, idrel/NP\{number:
pl\}]
\end{enumerate}

In English, the pronouns {\em this, that} must be adapted as well ({\em these,
those}), but in Dutch they remain {\em dit,dat }. A similar phenomenon can 
be seen in English {\em there } and Dutch expletive {\em  er}.

\end{itemize}
 

\section{ Other pronouns}
In this section the treatment of some further pronouns is discussed:
\begin{itemize} 
  \item I propose to treat {\em  dezelfde, diezelfde, datzelfde, ditzelfde,
hetzelfde}
as {\em de } + GLUE + {\em zelfde} etc. This is the easiest
way to account for their distribution. Cf. also {\em een zelfde }, {\em  deze
zelfde} (not written as one word!). Of course the category of {\em zelfde }
must now be determined. Van Dale calls it an adjective. I doubt whether this
is the most appropriate choice. There is also the use of {\em 
dezelfde } as an adjective?, as in {\em twee dezelfde boeken} (not in Van dale).
The relevant words can be 
used adjectivally (e.g. {\em  dezelfde man}), but also substantivally ({\em 
dezelfde}). I assume that in the latter case there is an empty head (EN).

  \item {\em niet }. It occurs only in some fixed expressions, e.g. {\em  om 
niet, voor niet}. Another example of its use in the Van Dale is misplaced
(it should be under the adverbial {\em niet } (NEG in R3): {\em dat is niet
meer dan een suggestie }. It has code 48.  I propose to remove it, and to
add the fixed idioms {\em  voor niet, om niet}
  \item {\em het } The treatment of {\em het } in Van Dale is totally
inconsistent. A distinction is made between {\em het I } of category 
PERSPRO (code 41) and {\em  het II} of category 'onbepaald voornaamwoord'
(code 48). It looks as if the distinction corresponds to referential
{\em het } (e.g. {\em het liep naar huis}) vs. nonreferential {\em  het}
(e.g. {\em  Hij betreurde het, dat ..}, but if so, then the distinction has
not been made consistently. I propose to drop this distinction. In R3 the
distinction will be made by the rules. Argument substitution rules
introduce referential {\em  het}, other rules (e.g. extraposition rules)
introduce nonreferential {\em het} syncategorematically. 
   \item {\em er } in one of its meanings is called a personal pronoun, with
code 41. ( 'oude tweede naamval van de derde pers. mv.'). This is nonsense.
This {\em er} is what in R3 will be called {\em  quantificational er }, of
category RADV. 
  \item {\em een} The treatment of {\em een} is a total mess in Van Dale.
      A special section (see below) is devoted to this word. 
  \item {\em  idem} is called an indefinite pronoun by Van Dale. As far
as I can see there are two ways of using this word: First, as a sort of
adverb, in a way similar to {\em ook }, cf. e.g. {\em  Piet is naar huis
gegaan, en Klaas idem}. Second, it can be used in lists to repeat material,
, e.g. to repeat the preceding author in a reference list. 
I propose to disregard the second way of using it, and to assign the category
ADV to it.
   \item {\em ander} is called an indefinite pronoun by Van Dale. However,
  given such forms as {\em een ander, de ander, geen ander} vs. e.g.
{\em *een iedereen/iemand, de iedereen/iemand, geen iedereen/iemand } etc.
this is not so clear. I propose to consider the following elements INDEFPRO-s:
{\em een ander, geen ander, de ander, het ander } (as (fixed) idioms)(Also
{\em de een, het een }, see below). Of
course, {\em ander} can be used adjectivally as well. 
   \item {\em  eentje} is called an indefinite pronoun. This is nonsense.
In R3 it will be a BNUM, although of special kind, that can occur only with
EN(count) as a head.
   \item {\em welk II } is a relative pronoun, and it is also 
used in such constructions as: {\em  Zij verkochten koffie en thee, {\em welke
 } artikelen erg in trek waren}, where it might be called a RELADJ. 
This use will not be dealt with in R3
   \item {\em zelf} is called a 'aanwijzend voornaamwoord' in Van DALE. I
        propose to assign the category BADV to it in Rosetta3 
\end{itemize}

\section {The treatment of {\em  een} in Van Dale}

I will ignore {\em een}$^{1}$ (as a substantive). 

W.r.t {\em een}$^{2}$ the following can be remarked: Under I the use of {\em
een } as an adjective is dealt with (as in : {\em een worden, zijn} and {\em
een met }. I have no comments except for the fact that I doubt whether {\em
adjective } is the most appropriate category here. 
Under II  its use as a "bijwoord; tot bijwoord geworden lidwoord" is treated.
It has two submeanings: {\em ongeveer}, and {\em zeer veel}.
Examples of the first submeaning are: 
\begin{itemize}
  \item {\em  een dag of wat; een uur of drie}
  \item {\em een veertien dagen; een duizend gulden }
\end{itemize}

The first examples seem nonsense to me. The {\em  ongeveer} reading is caused
by {\em een } in combination with {\em  of[ wat], of [drie]}. In these case
the {\em een } is simply a BART. The second examples
are real examples.
The second submeaning is illustrated by:
\begin{itemize}
  \item {\em  het kost me een geld; een mensen dat daar waren; het was daar 
een temperatuur}
\end{itemize}
This use seems to be the same as the use of {\em 'n } (listed in VanDale only
as an article), and in fact they seem to be spelling variants. I propose
to consider this use of {\em een} the same as the use of {\em 'n }, and to 
assign it the same category, whatever this will be (e.g. EXCLAMADJ?).

The treatment of {\em een}$^{3}$ ("onbepaald voornaamwoord")is the most messy
one of all. 
Two submeanings are distinguished:
\begin{enumerate}
  \item {\em iemand, iets }
  \item {\em een of ander, een zekere }
\end{enumerate}
The first submeaning is illustrated by the following examples:
\begin{itemize}
  \item {\em op een dag }. This is totally misplaced. It should be an 
illustration of the second submeaning
  \item {\em er is er een voor u geweest; elke stem is er een }. These 
examples are misplaced as well. They illustrate the use of the 
\underline{numeral} 
{\em een } in combination with quantificational {\em er }
  \item {\em dat is een!} This example is misplaced. It illustrates the use
of the \underline{numeral} {\em een } in an expression.
  \item {\em er een laten vliegen, er een pakken,nemen, drinken }. These 
illustrate the use of {\em  een} as \underline{numeral} in combination with 
quantificational {\em er } in (idiomatic) expressions.
   \item {\em niet een, geen een} These examples as illustrating submeaning
1 are totally misplaced. These examples can be used adjectivally only. I 
propose to consider these cases as combinations of {\em niet } or {\em geen}
with the numeral {\em een}, where {\em geen een } is a variant of 
{\em niet een }. Notice, that quantificational {\em er } must appear
when these forms are used substantivized.
   \item {\em een van hen, een van twee\"{e}n}. These illustrate {\em een }
used as a numeral, in my opinion (cf. {\em  twee van hen, twee van de drie} 
etc).
   \item {\em van het een op het ander, een-twee-drie } The first example
is an idiomatic expression. Notice that also {\em het een, het ander } must
be distinguished (I propose to call them INDEFPRO) See also above, under
{\em  ander}. The second example is a (fixed) idiom, {\em een-twee-drie}, or
(preferably) {\em zo een-twee-drie }, meaning (approximately)
 {\em zo plotseling, zo opeens}. It seems to me that it could better be 
rubricized under {\em een } as a numeral.
   \item {\em zij is een-en-al glimlach, een of ander, op de een of andere 
wijze }. The expressions {\em een-en-al, de een of ander, het een of ander, het
een en ander} and {\em een of ander } are (fixed) idioms, of category BADV,
INDEFPRO, INDEFPRO, INDEFPRO, and BQ or BDET or whatever, resp.. 
   \item {\em als er een is die het kan, dan is hij het } This is a case
of an (idiomatic?) expression where the \underline{numeral} {\em een } has been used 
in combination with quantificational {\em er }
\end{itemize}
The second submeaning is illustrated in the following way:
\begin{itemize}
  \item {\em Woont hier ene Jansen?} This use of {\em ene}, also
       sometimes as {\em een } (cf. {\em een meneer Jansen })is a real example 
of adjectival pronominal use.         This use is not dealt with in this note.
See next section. 
  \item {\em je bent me er (ook) een }. This is an example of an expression
  where the numeral {\em  een} has been used in combination with 
quantificational {\em  er}.
\end{itemize}

{\em een}$^{4}$ is subdivided into three submeanings:
\begin{enumerate}
  \item 'hoofdtelwoord'
  \item 'rangtelwoord'
  \item 'lidwoord'
\end{enumerate}
 
I will ignore the use as a 'rangtelwoord'. This is nonsense, but the treatment
here is the same as the treatment of all other 'hoofdtelwoorden', so this 
should be dealt with separately.

The first submeaning has been illustrated by the following examples:
\begin{itemize}
  \item {\em het een en ander, een keer is voldoende, in een woord}
      The first example is misplaced. See above. 
  \item {\em een en dezelfde} This is a (fixed) idiom.
  \item {\em de weg is een en al modder}. {\em een en al } is a (fixed) idiom.
      See also above.
  \item {\em niet een heeft er iets over gezegd }. See above. The 
quantificational {\em  er} and prepositional {\em er } coincide here.
  \item {\em  het is bij enen, honderd tegen een}. These are correct examples.
The {\em -en} forms must be accounted for. Harm Smits and my proposal is to 
list these forms in the lexicon, until/unless a grammar for numeral expressions
is made. The category BNUM will need an attribute {\em enform:BOOLEAN }. 
  \item {\em men kan het ene doen en het andere niet laten }. This is an 
idiomatic expresssion. {\em  het ene, het andere} must be INDEFPRO-s
   \item {\em  een voor een, als een man, bij hem is er een op de loop} 
Correct examples. 
\end{itemize}

In submeaning III, the following examples have been given:
\begin{itemize}
  \item {\em een ander}. Misplaced example
  \item {\em een meneer A } Misplaced example. See above.
  \item {\em een paard, hij is een Picasso, een Rembrandt, neem een Tedje van
       Es, een vrouw}. Correct examples (partially with a slightly different 
meaning than usually)
   \item {\em  wat (voor) een, zo een} These are BDET-s, or BQ-s. Not dealt
with in this note.
\end{itemize}

\section{ A proposal for {\em een }}

\begin{description}
  \item[een$^{1}$] Spelling variant: {\em \'{e}\'{e}n}. Category: BNOUN.
         This corresponds to Van Dale  een$^{1}$
  \item[een$^{2}$] Spelling variant: {\em \'{e}\'{e}n}? Category: BADJ.
         This corresponds to Van Dale een$^{2} I$
  \item[een$^{3}$] Spelling Variant: {\em 'n }. Category: BADV. cf.
{\em een veertien dagen, een duizend gulden}. It means 'approximately' and
modifies cardinals. Compare also {\em zo'n } in e.g. {\em zo'n duizend gulden}
  etc. It corresponds to Van Dale {\em een$^{2}$ II 1}
   \item['n] Spelling variant {\em een}. Category: BDET? It corresponds to 
               Van Dale {\em een$^{2}$ II 2 }. It means approximately the same
as {\em wat 'n} (but it is not wh), and is used in exclamations.
   \item[de een, het een, de ene, het ene] These are INDEFPRO's.
   \item[ene, een] as in {\em ene Jansen, een meneer Jansen}. Category: BDET?
      also appearing in such expressions as {\em op een dag, op een ochtend}?
   \item[een$^{4}$] Spelling variant: {\em \'{e}\'{e}n}. Category: BNUM. It 
corresponds to Van Dale {\em een$^{4}$ I,II}.
   \item[een$^{5}$] Spelling variant: {\em  'n}. Category: BART. It corresponds
to Van Dale {\em  een$^{4}$, III}.
\end{description}

\section{Appendix: List of all pronouns}

In this appendix all pronouns mentioned in the text are listed, and an
indication concerning how they should be treated according to this proposal.
is given. The Roman numbers refer to subclassifications made in the Van Dale.
