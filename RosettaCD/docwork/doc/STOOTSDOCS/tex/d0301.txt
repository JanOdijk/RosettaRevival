\section{Introduction}
The CLAUSE grammar of Spanish is divided into the following parts:\\
\begin{itemize}
\item VERBderivation (see doc:r0349)
\item VERBPPROPformation
\item XPPROPtoCLAUSE (see doc:r0334)
\item CLAUSEtoSENTENCE (see doc:r0328)
\item UTTERANCE (see doc:r0344)
\end{itemize}
This document discusses the subgrammar VERBPPROPformation for Spanish.
In section 2 a specification of the subgrammar is given (i.e. the Head,
Import and Export Categories). 
In section 3 the rules and transformation are listed in the so-called
Control Expression.
In section 4 documentation on the rules and transformations is given.
Each subsection contains a rule class or transformation class.\\
In principle, the rules/transformations have been documented by their
own author: Andr\'{e} Schenk wrote the sections on TC\_IdiomCheck and
RC\_IDstartVerbpRules. Lisette Appelo wrote the sections on 
TC\_AktionsartCalculation
and RC\_TempAdvVar. The other sections have been written by Elly van Munster.

\section{Subgrammar Specification}
\begin{description}
  \item[Head] SUBVERB
  \item[Export] VERBPPROP
  \item[Import] NPVAR, CNVAR, ADJPPROPVAR, ADVPPROPVAR, NPPROPVAR, 
                PREPPPROPVAR, VERBPPROPVAR, SENTENCEVAR, CLAUSEVAR, 
                EMPTYVAR, OPENADVPPROP, ADVPVAR,  PREPPVAR, ADVPVAR, 
                PREPPVAR


  \item[File] spanish:verbppropformation.mrule (mrules28.mrule)
\end{description}

\section{Rules and Transformations}

\begin{mruleclass}{TC\_IdiomCheck}
\begin{classdescr}
\kind Obligatory Transformation class
\classtask To determine whether the skey of the head (SUBVERB) is an idiom key 
or not. In case it is not, the ordinary startrules apply, in case it is, the 
idiom start rules are called. These transformations are used because otherwise 
the "ordinary" startrules might apply to an idiom verb, resulting in a crash in 
morphology since an idiom does not have a string directly connected with it.\\
For more information see doc.273 on idioms.
\end{classdescr}

\begin{members}
\begin{member}
\rulename TIsIdiom
\ruletask To check whether the skey of the SUBVERB is an idiom key. If it is, 
the special idiom startrules are called, and the ordinary startrules are 
skipped (see control expression).
\file spanish:RC\_idstartverbprules.mrule (mrules107.mrule)
\semantics \nosemantics
\example {\em pegar} un puntapi\'{e} a alguien (Dutch: 'iem. een trap geven')
\remarks\mbox{}

\end{member}
\begin{member}
\rulename TIsNotIdiom
\ruletask To check whether the skey of the head (SUBVERB) is not an idiom key. 
If it is not, the special idiom startrules are skipped, and the ordinary 
startrules can apply (see control expression). 
\file spanish:RC\_idstartverbprules.mrule (mrules107.mrule)
\semantics \nosemantics
\example {\em pegar} a alguien (Dutch: 'iemand slaan')
\remarks\mbox{}

\end{member}
\end{members}

\nofilters

\nospeedrules

\noplannedrules

\norulesnotince

\end{comments}

\end{mruleclass}


\begin{mruleclass} {RC\_StartVerbpRules}
\begin{classdescr}
\kind Obligatory Rule Class
\classtask To provide a (non-idiom) subverb with its correct number of argument
variables and build a VERBP and VERBPPROP around it. There are rules for 
ordinary verbs and rules for semi-idioms. The import (i.e. the argument(s) the
verb takes) to the former rules may be variables of any kind. The import to the
latter rules is subjected to an extra check.\\
The subverb is changed into a VERB, with all new attributes still at default 
value. \\ 
{bf NB.} In Spanish only one startrule has been written which switches 
arguments (Rstartvpprop021). Maybe it will be necessary to add more of these 
rules later on (e.g. Rstartvpprop132, etc..)
\end{classdescr}

\begin{members}
\begin{member}
\rulename RSTARTVPPROP000
\ruletask 
To build a VERBP and a VERBPPROP around a SUBVERB with thetavp000:
a subjrel/"AUX\_NPONredelloBPRSPR" is made and the VERBP is provided with
no argument variables.
\file spanish:rc\_startverbprules.mrule (mrules13.mrule)
\semantics \nosemantics
\example llueve, nieva, graniza, etc.
\remarks\mbox{} 

\end{member}
\begin{member}
\rulename RSTARTVPPROP100

\semantics \nosemantics

\begin{mruleclass}{RC\_IdStartVerbpRules}
\begin{classdescr}
\kind Obligatory Rule Class
\classtask On the basis of a syntactic derivation tree that is specified in 
the M-rule  model the syntactic S-tree for an idiom is built. This includes 
much more than just a VERBP and a VERBPPROP. For further explanation, see doc.\ 
273 on Idioms.


\end{classdescr}

\begin{members}
\begin{member}
\rulename RSIDSTARTVPPROP120
\ruletask To combine a SUBVERB with its required number of arguments and 
to put one of them in subject position and one inside the VERBP; the head of 
the argument NP in VERBP is specified; this is a part of a semi-idiom.
\file spanish:rc\_startverbprules.mrule (mrules13.mrule)
\semantics \nosemantics
\example x1 da grito, Mar\'{\i}a da un grito
\remarks\mbox{} 
\item comment:\\
In this rule the subverbrecords are copied from each 
other analytically and generatively; another way is to change the bverb first
(i.e. attributes values are the same except for the key) and then copy the
subverb from the bverb. If dictionary structure changes: keep this in
account!!! 

\end{member}
\begin{member}

\file spanish:rc\_startverbprules.mrule (mrules13.mrule)
\semantics \nosemantics
\example x1 da demonstraci\'{o}n a x2, Mar\'{\i}a da una demonstraci\'{o}n a
Pedro.
\remarks\mbox{} 
\item comment:\\
In this rule the subverbrecords are copied from each

\begin{mruleclass}{TC\_Verbpatternrules}
\begin{classdescr}
\kind Obligatory Transformation Class
\classtask To check the categories of the argument variables and the relation 
they bear in the VERBP against the verbpatterns specified for the verb.
The synvpefs attribute of the VERBPPROP is set at the value actually chosen 
from the synvps of the VERB.
\end{classdescr}

\begin{members}
\begin{member}
\rulename TVerbpattern0
\ruletask To let VERBPPROPs  with a head VERB with synvps = [noVPargs] 
pass the verbpattern-transformations.
\file spanish:TC\_vp1.mrule (mrules30.mrule)
\semantics \nosemantics
\example $<$empty$>$ llueve $\rightarrow$ $<$empty$>$ llueve \\
Juan duerme $\rightarrow$ Juan duerme
\remarks\mbox{}

\end{member}
\begin{member}
\rulename TVerbpattern1
\ruletask Verbpattern-transformation for verbs with 
{\em one non-prepositional} argument in the VP.

\semantics \nosemantics
\example 
\begin{description}
   \item[subrule 1] \'{E}l pega a Juanita; Pedro ven\'{\i}a. (synNP)
   \item[subrule 2] Pedro come (x2). (synEMPTY)
   \item[subrule 3] x1 considera [Pedro el mejor presidente]. (synCLOSEDNPPROP)\\
    salir [x1 un c\'{e}lebre m\'{e}dico]. (synOPENNPPROP)
   \item[subrule 4] yo considero [Juan inteligente];\\ 
                    quedar(se) [Pedro ciego]. (synCLOSEDADJPPROP)
   \item[subrule 5] esto es (= smaakt) [x1 rico]. (synOPENADJPPROP)
   \item[subrule 6] esto cuesta cinco florines. (synMEASUREPHRASE)
   \item[subrule 7] $<$empty$>$ parece que est\'{a} enfermo. (synCOMOCLOSEDADJPPROP)
   \item[subrule 8] (Dutch: ik [hem uit de box] kreeg) \\
                    (Dutch: ik [hem daar] kreeg). (synLOCCLOSEDPREPPPROP)\\ 
                    Spanish: no example found yet.
   \item[subrule 9] (Dutch: ik [hem de box uit] kreeg). (synDIRCLOSEDPREPPPROP)\\
                    (In Sp. always a preposition!) 
   \item[subrule 10] x1 vive [ x1 en Amsterdam]. (synLOCOPENPREPPPROP)
   \item[subrule 11] x1 entra [ x1 por la puerta]. (synDIROPENPREPPPROP)
   \item[subrule 12] x1 tiene [los labios pintados]. (synCLOSEDVERBPPROP)
   \item[subrule 13a] x1 quiere [x1 venir]. (synOPENINFSENT)
   \item[subrule 13b] x1 tiene [que x1 venir]. (synQUEOPENINFSENT)
   \item[subrule 13c] x1 hay [que x1 venir]. (synQUECLOSEDINFSENT)
   \item[subrule 14] x1 ve [Pedro venir]. (synCLOSEDINFSENT)
   \item[subrule 15] x1 lamenta [el tener que partir]. (synELOPENINFSENT)
   \item[subrule 16] x1 lamenta [el que tenga que partir]. (synELQUESENTsubj)
   \item[subrule 17] x1 huele [x1 bien]. (synOPENADVPPROP)
   \item[subrule 18] x1 dijo [que x2 ven\'{\i}a]. (synQUESENTind)\\
                     x1 quiso [que x2 viniera]. (synQUESENTsubj)
   \item[subrule 19] x1 sabe [si pod\'{\i}a venir]\\
                     x1 sabe [qu\'{e} deb\'{\i}a hacer]. (synQSENT)
   \item[subrule 22] yo me lo pregunto. (PROSENT spelled out as {\em lo})\\
                     (synPROSENT)
   \item[subrule 23] llevar [x1 trabajando por tres horas]. (synOPENGERSENT)
   \item[subrule 24] caer [esto en desuso];\\
                     caer [esto ADVP??]. (synOTHERCLOSEDPREPPPROP)
\end{description}

case the information could be maintained in which voice (active or passive) the

\begin{verbatim}
IL-active   ----->>   Sp.1) active

IL-passive  ----->>   Sp.1) SEpassive

(SE) come (SUBRULE 2).
\remarks\mbox{}

Condition on "Human" added in SEpassive.\\
In SEactive2 a condition added, such that PERSPRO's in objrel are admitted, too.
NB. Note that PERSPRO's cannot be tested on Human; only a test on genero, such 
that:\\
 - Hij wordt geslagen. admits SEactive2 ("hij" is non-neutro).\\
 - Het wordt geslagen. doesn't admit SEactive2 ("het" is neutro).\\
NB. Note that this is not correct in case of a text-translation: if "hij" is
non-Human, SEactive2 shouldn't apply, i.e. a PERSPRO should have a condition on
