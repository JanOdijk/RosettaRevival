\section{Introduction}

This document describes the attributes of the BVERB record, and their 
interpretation.

It also serves as a guide for dictionary filling.

Several appendices with relevant information for filling the dictionary have
been added.

Appendix D has been written by Andr\'{e} de Vos, and is a report of the 
dictionary filling (for verbs) done by him. 

Some of the appendices have been written in Dutch. 

 
\section{The attributes of the BVERBrecord}

The attributes of the BVERB record are:
\begin{attributes}
      \att[req, env] These attributes are used to indicate the 
         {\em  polarity} properties of the verb. The attribute is a
set attribute, but only for historical reasons. Sensible values are
\begin{values}
\val[{[pospol, negpol, omegapol]}] The default value, indicating that no
relevant polarity properties hold.
\val[{[pospol]}] The verb requires (req) or creates (env) a positively polar
environment
\val[{[negpol]}] The verb requires (req) or creates (env) a negatively polar
environment
\end{values} 

The following tests can be used to determine polarity properties.

If a verb can replace V in the following sentences, then env={\em [negpol]}:

\begin{description}
  \item[] {\em Hij V een hand uit (te) steken} (idiomatic reading!!)
  \item[] {\em Hij V ook maar iets (te) doen }
\end{description}

(example: {\em  weigerde}).

If a verb can replace V in the first but not in the second context, then
req={\em [negpol]}:

\begin{description}
  \item[] {\em Hij V niets (te) doen  }
  \item[] {\em Hij V iets  (te) doen}
\end{description}

(example: {\em  hoeft}). If the verb cannot replace V in both contexts, nothing
can be said yet. A similar test is formed by the following two contexts:

\begin{description}
  \item[] {\em Niemand V (hem/het)}
  \item[] {\em Iemand V (hem/het)}
\end{description}



 

      \att[conjclasses] This attribute indicates the conjugation classes of
the verb. (filled automatically)
      \att[particle] This attribute contains the key of the particle, if the
verb has one, otherwise zero. (filled automatically)
   \att[possvoices] This settype attribute indicates which voices the verb can
be in. 
The values that can appear in the set are:
      \begin{values}
	\val[Active] Active voice, ex. {\em  hij koopt het boek }
	\val[Passive] Passive Voice, ex {\em  het boek wordt gekocht},
                       {\em Er wordt gedanst }
        \val[AanActive] AanActive Voice, e.g. in  {\em (Ik liet)  NP aan NP V}
e.g. {\em (Ik liet) het aan mijn broer lezen}.The verbs {\em lezen, horen,
zien, voelen, ruiken, proeven} (and only these!) can have this value in their
attribute possvoices (of type Voicesettype). 
Sentences in this voice can appear only after {\em laten}. 
       \val[DoorActive] DoorActive is a value that appears in the bracketed
part of 'Ik liet [NP door NP V]', e.g. {\em [Ik liet] het huis door de man
schoonmaken}. It appears to be the case that any verb that allows passive also
allows DoorActive except for intransitives and verbs that allow AanActive. 
      \end{values}



   \att[reflexivity] This attribute indicates whether the verb is reflexive,
reciprocal, or none of these. 
  The attribute has three possible
values: 
      \begin{values}
	\val[notreflexive]the verb is neither reflexive nor reciprocal (ex.
{\em  kopen}) 
	\val[reflexive] the verb is reflexive; It requires {\em zich} to appear
(ex. {\em  schamen}). The attribute {\em  reflexive} incorporates both
cases of so-called {\em  toevallig wederkerende werkwoorden} (example:
{\em  zich scheren}) and cases of so-called {\em noodzakelijk wederkerende
werkwoorden} (example: {\em  zich schamen})
        \val[reciprocal] the verb requires a reciprocal pronoun 
          ({\em elkaar}).     
  Ex. {\em  tegenkomen} in {\em elkaar tegenkomen}, cf. the English translation
{\em  They met}
    \end{values}

   \att[synvps] See the document on verbpatterns
   \att[thetavp] Used to indicate the number of arguments and their 
partitioning (inside VP or outside VP). The following values are distinguished:
\begin{values}
\val[vp000] Verb not taking any arguments, e.g. {\em  regenen, sneeuwen, 
ijzelen}
\val[vp010] Ergative Verbs taking one argument, e.g {\em  vallen, aankomen,
                sterven}
\val[vp100] Intransitive Verbs, e.g. {\em  dansen, werken, slapen}.
\val[vp120] Dyadic verbs, a.o. transitive verbs, e.g. {\em  zien, bouwen,
             doden, proberen, geloven}
\val[vp012] Dyadic ergative verbs, e.g. {\em  opvallen, meevallen, ontgaan}

\val[vp123] Ditransitive verbs, e.g {\em  geven, verkopen, beschuldigen} 
\val[omegathetavp] a default value; of no significance for BVERB
\val[thetavpaux] a special value for auxiliary verbs.
\end{values}

   \att[CaseAssigner] This attribute indicates whether the verb 
 is a Case Assigner
or not. It is used in combination with {\em thetavp} and {\em reflexivity}
 to subclassify 
certain verbs. In particular:
\begin{description}
\item[] \mbox{}\\
\begin{tabular}{|c|c|c|c|}
\hline
Thetavp    & Reflexivity  & CaseAssigner  & Example Verb(s)\\
\hline
\hline
vp010      & false        & false         & aankomen (ergative verbs)\\
vp010      & false        & true          & het {\em regent} verwijten\\
vp010      & true         & true          & zich voordoen\\
vp010      & true         & false         & does/can not exist\\
vp012      & true         & true          & zich openbaren\\
\hline
\end{tabular}
\end{description}
Intransitive and transitive verbs have this attribute set to {\em  true},
Ergative verbs (1-place and 2-place) have this attribute set to {\em false}.
Verbs taking no arguments have this attribute set to {\em  true}.

   \att[subc] used to subclassify verbs. The values are:
\begin{values}
\val[Modalverb] for the verbs {\em  willen, mogen, kunnen, moeten, zullen}
   and these only.
\val[progaux] for verbs like {\em  zitten(te), staan (te)} etc. in 
constructions such as {\em  hij zit te kaarten}, {\em  hij staat te lummelen}
\val[hebbenaux] for the auxiliary verb {\em  hebben}
\val[zijnaux] for the auxiliary verb {\em  zijn}
\val[latenverb] for the verb {\em  laten}
\val[wordenaux] for the auxiliary verb {\em  worden}
\val[Mainverb] for all other verbs
\end{values}

   \att[perfauxs] This attribute indicates what auxiliary verbs are used in
the perfect: {\em hebben} (value: [hebaux]), {\em zijn} (value: [isaux]), 
or both (value:[hebaux, isaux]).
   \att[prepkey1, prepkey2] These attributes indicate the keys of the 
prepositions used in the prepositional object(s). 
There are two attributes, for cases such as {\em met iemand
over iets spreken} cf. English {\em to speak to someone about something}. 

   \att[controller] This attribute indicates what the controller of the subject
of an embedded infinitival complement is for this verb.
The values are:

      \begin{values}
	\val[subject] The subject is the controller
	\val[object] The object is the controller
        \val[indobj] The indirect object, or the NP in aanobjrel or voorobjrel
                        is the controller
        \val[prepobj] The object of the prepobjPP is the controller
        \val[none] No controller restriction is specified
      \end{values}

The relation mentioned is the relation before application of object-to-subject
transformations have applied (in passives, ergatives).

Tests to determine controllerhood: If a verb can replace V in 

\begin{description} 
  \item[] {\em hij V mij zich te gedragen}
  \item[] {\em hij V mij zich te scheren}
  \item[] {\em hij V mij z'n geduld te bewaren}
\end{description}

\noindent
then controller=subject.

If a verb can replace V in 

\begin{description}
  \item[] {\em ik V hem zich te gedragen}
  \item[] {\em ik V hem zich te scheren}
  \item[] {\em ik V hem z'n geduld te bewaren}
\end{description}
 
then controller = object, or indobj, depending on the relation of {\em  hem}

Similar tests can be constructed for controllers in prepositional objects.
   \att[verbraiser] 
Used to indicate whether a verb allows, 
requires or disallows Verb Raising.

The values are:
\begin{values}
  \val[optionalVR] The verb allows Verb Raising
  \val[obligatoryVR] The verb requires Verb Raising
  \val[noVR] The verb disallows Verb Raising
\end{values}


A test to determine verbraiserhood (for verbs with thetavp=vp120, and taking
open sentential complements): 
If a (finite) verb can be inserted 
instead of V in

\begin{description}
\item[S1] {\em ..omdat hij het boek V (te) lezen}
\end{description}
\noindent
then this verb is either 

{\em optionalVR} or
{\em obligatoryVR}. If in addition the verb requires {\em  te} and 
can be inserted instead of V in
\begin{description}
\item[S2]  {\em ..omdat hij V het boek te lezen}
\end{description}
\noindent
it is {\em optionalVR}, otherwise 
{\em obligatoryVR}. If it cannot appear in the first context sketched it is 
{\em NoVR}.

Verbs taking closed sentential complements are obligatory Verbraisers.

For verbs with {\em  thetavp=vp123} the following contexts can be used 
analogously:

\begin{description}
\item[S1] {\em ..omdat hij de jongen het boek V (te) lezen}
\item[S2] {\em ..omdat hij de jongen V het boek te lezen}
\end{description}

For verbs with thetavp=vp012 the following contexts hold:
\begin{description}
\item[S1] {\em ..omdat hem het boek V (te) lezen}
\item[S2] {\em ..omdat hem V het boek te lezen}
\end{description}

If a verb allows {\em  te}, but does not (always) require it, then you must
usually split it up into two different verbs.


Examples:
\begin{description}
  \item[thetavp=vp120, open S-compl]\mbox{}\\
\begin{description}
  \item[Proberen] {\em ..omdat hij het boek probeerde te lezen} and
                  {\em ..omdat hij probeerde het boek te lezen} are both 
correct, hence {\em optionalVR}
   \item[willen] {\em ..omdat hij het boek wilde lezen} correct, the verb does 
not allow a complement with {\em  te}, hence {\em obligatoryVR}
   \item[afspreken] {\em {\star} ..omdat hij het boek afsprak te lezen}, hence
{\em noVR}. 
\end{description}
\item[thetavp=vp123]\mbox{}\\
\begin{description} 
  \item[leren, no {\em  te}]  {\em ..omdat hij de jongen het boek leerde lezen},
is correct. The verb does not allow {\em  te} (cf. {\em {\star}..omdat hij de 
jongen het boek leerde te lezen}), hence {\em  obligatoryVR}
  \item[leren, with {\em  te}]
{\em {\star}..omdat hij de jongen het boek leerde  te lezen}, hence {\em noVR}
  \item[opdragen] {{\star}..omdat hij de jongen het boek opdroeg te lezen},
hence {\em noVR}
\end{description}
  \item[thetavp=vp012]\mbox{}\\
\begin{description}
  \item[meevallen] {\em  {\star}..omdat hem het boek meeviel te lezen}, hence
{\em noVR} (cf. {\em ..omdat het hem meeviel het boek te lezen})
\end{description}
\end{description}

Summarizing, in all tests two sentences are used, called S1 and S2.

\begin{description}
  \item[]\mbox{}\\ 
\begin{tabular}{|c|c|c|}
\hline
can occur in S1 & can occur in S2 & verbraiser\\
\hline
no              &   irrelevant    & NoVR\\
yes             &   yes           & OptionalVR\\
yes             &   no            & ObligatoryVR\\
\hline
\end{tabular}
\end{description}



   \att[IPP] Used to indicate whether the verb participates in the phenomenon
of {\em  Infinitivum Pro Participio} (Infinitive instead of Participle).

If a verbraiser is put in a perfect tense, sometimes an infinitive
can or must appear instead of the expected participle if it is followed by an
infinitive, e.g. {\em  {\star} Hij heeft
{\bf gewild} komen} vs. {\em  Hij heeft {\bf willen} komen};
{\em  ?Hij heeft het boek {\bf geprobeerd} te lezen} vs. {\em Hij heeft
het boek {\bf proberen} te lezen}.
The values are:
\begin{values}
\val[ObligatoryIPP] The verb participates in IPP obligatorily
\val[OptionalIPP] The verb participates in IPP obligatorily
\val[NOIPP] The Verb does not participate in IPP, or nothing can be said
         about it.
\end{values}


The following test can be used:

Check whether a verb can replace V in one of the following two sentence as a
past participle or as an infinitive:

\begin{description}
  \item[] {\em  hij heeft het boek V (te) lezen}
  \item[] {\em Hij heeft Piet het boek V (te) lezen}
\end{description}

The following table then yields the correct value for IPP:

\begin{description}
  \item[] \mbox{}\\
\begin{tabular}{|c|c|c|}
\hline
V=inf OK & V=pastpart OK & IPP\\
\hline
no       &  no           & noIPP\\
no       &  yes          & noIPP\\
yes      &  no           & obligatoryIPP\\
yes      &  yes          & optionalIPP\\
\hline
\end{tabular}
\end{description}


Examples: (no,no) {\em kopen}; (no,yes) no known examples;
(yes,no): {\em zien, willen}; (yes,yes) {\em  ?proberen, ?beloven}; 


   \att[classes]  Used to partition verbs into (semantic) classes
w.r.t. the type of event described. It is used in TIME rules (to
compute {\em  aktionsart}. the values are:
\begin{values}
\val[stativeclass]
\val[movementclass]
\val[durativeclass]
\val[momentaryclass]
\val[iterativeclass]
\end{values} 

For explanation of these values I refer to Lisette Appelo.

   \att[KEY] This attribute indicates the key of the BVERB
\end{attributes}

\section{Consistency Checks}

The following consistency checks are currently performed:

\begin{verbatim}

< BVERB
:  IMPLIES((particle <> 0), (verbraiser = noVR))    
            "Particle and verbraiser mutually exclusive"           
:  IMPLIES((reflexivity=reflexive), (verbraiser = noVR)) 
            "Reflexive verbs are no verbraisers"
:  IMPLIES((reflexivity=reflexive), (perfauxs = [hebaux]))     
            "reflexive verbs are have [hebben] as perfaux"
:  IMPLIES((IPP IN [obligatoryIPP, optionalIPP]),
                   (verbraiser IN [obligatoryVR, optionalVR]))           
            "IPP is only relevant for verbraisers"
:  IMPLIES((controller = subject), (thetavp IN [vp120, vp123]))
            "subject controller requires vp120 or vp123"
:  IMPLIES((controller = object), (thetavp IN [vp012, vp123])) 
            "object controller requires vp012 or vp123"
:  IMPLIES((controller = indobj), (thetavp IN [vp123, vp012])) 
            "indobject controller requires vp123 or vp012"
:  IMPLIES((subc = modalverb), (IPP = obligatoryIPP))          
            "modalverbs must have IPP=obligatoryIPP"
:  IMPLIES((reflexivity IN [reflexive, reciprocal]),
                   ((possvoices*[Passive, DoorActive, AanActive]) = [])) 
            "reflexive and reciprocal verbs can appear in active only"
:  IMPLIES((synvps * LSAUXDOM_oblVRvps <> []),
                   (verbraiser = obligatoryVR))                          
            "verbpattern requires obligatoryVR"
:  IMPLIES((verbraiser = obligatoryVR),(IPP = obligatoryIPP)) 
            "obligatory verbraisers have IPP=obligatoryIPP"

           (synvps * LSAUXDOM_VRvps <> [])
          )           
            "verbraisers require an appropriate verbpattern"
:  IMPLIES((thetavp=vp000), (synvps <= LSAUXDOM_vp000vps ))       
            "synvps inconsistent with thetavp"
:  IMPLIES((thetavp=vp100), (synvps <= LSAUXDOM_vp100vps ))       
            "synvps inconsistent with thetavp"

            "synvps inconsistent with thetavp"
:  IMPLIES((thetavp=vp120), (synvps <= LSAUXDOM_vp120vps ))      
            "synvps inconsistent with thetavp"

            "synvps inconsistent with thetavp"

            "synvps inconsistent with thetavp"

\section{Suggestions for filling the dictionaries}

The attributes {\em conjclassses, particle, perfauxs} have been filled 
correctly  automatically.

The attribute {\em subc} is relevant only for  a limited number of verbs
that have very special syntactic behavior anyway. I would propose not to
fill these verbs normally, but set them apart. This implies that for
all other verbs the attribute {\em subc} has been filled correctly.

It seems best to me to fill the other attributes in a way that satisfies the
following order constraints ($\prec$ means `precede', `;' means `or'):

\begin{description}
\item[1] thetavp $\prec$ synvps 
\item[2] thetavp $\prec$  CaseAssigner; req; env; possvoices; reflexivity
\item[3] synvps $\prec$ prepkey1; prepkey2; classes; Controller; Verbraiser
\item[4] Verbraiser $\prec$ IPP
\end{description}

\section{Ergativiteit Criteria}

\begin{description}
  \item[hulpwerkwoord(zwak)] Indien een ww met {\em  zijn} vervoegd wordt, dan
                       is het ergatief. Indien het met {\em  hebben} vervoegd
                       wordt, dan is het niet ergatief. 
  \item[passivisatie (sterk)] Indien een ww gepassiviseerd kan worden, dan
                          is het niet ergatief. 
  \item[tweeplaatsige werkwoorden(sterk)] Indien in een bijzin het 
indirect object
    op kan treden voor het andere argument, dan is het werkwoord ergatief:
Voorbeeld:
 \begin{description}
  \item[] ..omdat {\em  de man} {\bf het boek} beviel 
\end{description}

Het indirect object argument {\em de man} kan in dit voorbeeld voorafgaan
aan het andere argument {\bf het boek}, dus is het werkwoord {\em  bevallen}
ergatief. 

\item[Thetarol(zwak)] Bij ergatieve werkwoorden draagt het argument de thetarol
{\em  thema}, wat impliceeert dat het (in een zeer abstracte en vage zin)
een `beweging' ondergaat.

\item[Thetarol(sterk, maar vaag)] Indien een argument van een werkwoord de
thetarol `Agens'draagt, dan is het werkwoord niet ergatief.

\item[attributief deelwoord(sterk)] als het passief deelwoord van een eenplaatsig
werkwoord kan optreden
in plaats van V in de context {\em  de V N}, waar bij de N een zelfstandig 
naamwoord is dat normaal als onderwerp van het werkwoord op kan treden, dan
is het werkwoord ergatief. Is dit niet mogelijk, dan is het werkwoord niet
ergatief. Voorbeelden:

\begin{description}
  \item[] {\em De jongen valt}, {\em De gevallen jongen},   dus {\em vallen}  
              is ergatief
  \item[] {\em  De boek zijn aangekomen}, {\em  De aangekomen boeken}, dus
              {\em aankomen} is ergatief
  \item[] {\em  De jongens dansten}, {\em *De gedanste jongens}, dus {\em  
dansen} is niet ergatief
   \item[] {\em  De mannen werken}, {\em  *de gewerkte mannen}, dus
{\em  werken} is niet ergatief.
\end{description}

\item[{\em -er/-aar}-formatie(sterk)] Indien van het werkwoord V een woord op
 {\em  -er} met de semantiek  {\em  iemand die (beroepshalve) Vt} afgeleid
kan worden, dan is het geen ergatief. Indien dit niet mogelijk is, dan is
het betreffende werkwoord waarschijnlijk ergatief.
\begin{description}
  \item[vallen] {\em *valler}, dus {\em  vallen} is waarschijnlijk ergatief
  \item[aankomen] {\em  *aankomer}, dus {\em  aankomen} is waarschijnlijk ergatief
  \item[dansen] {\em  danser}, dus {\em  dansen} is niet ergatief
  \item[werken] {\em  werker}, dus {\em  werken} is niet ergatief
\end{description}


\end{description}

\section{The treatment of {\em  het, er} and extraposed sentences}

Sentential arguments of Verbs can occur in principle as `subject-arguments',
as `complement'-arguments or as `prepositional object-arguments'. They
are usually realized syntactically in a special way. This document describes
the treatment of these phenomena in the Rosetta3 Grammar of Dutch.

\subsection{Sentential subject arguments}

Sentential subject arguments cannot be realized as subjects in the 
grammar of Dutch. A sentential argument that must function as a subject
can be realized in one of the following two ways:
\begin{description}
\item[leftdislocrel] (main clauses only) The sentence is put in a special 
relation called {\em leftdislocrel} (in the beginning of the sentence)
and the subject position 
is either empty or occupied by the DEMPRO {\em  dat}:
\begin{description}
  \item leftdislocrel/[Dat hij vervelend is](, subjrel/dat) verbaast mij  
  \item leftdislocrel/[dat hij ziek is]( subjrel/dat) impliceert dat hij niet 
aanwezig kan zijn.
\end{description}
Such sentences are impossible or very marginal in subordinate clauses:

\begin{description}
  \item[] *hij zei dat dat hij vervelend is(, dat) mij verbaast
  \item[] ?hij zegt dat dat hij ziek is(, dat) impliceert dat hij niet aanwezig kan 
zijn.
\end{description}
 
 
\item[extraposrel] The sentence is extraposed (to the end of the sentence)
and bears the relation {\em  extraposrel}.
In this case the subject position is occupied by the pronoun {\em  het}:

\begin{description}
  \item[] subjrel/Het verbaast mij extraposrel/dat hij vervelend is  
\end{description}

\end{description}

Notice that sentential subject arguments can occur only with verbs that have
thetavp set to  vp100, vp120, or vp123.
Typical examples: {\em  verbazen, ergeren, interesseren, boeien}

The same type of behavior is found with sentential subject arguments of
adjectives.

\subsection{Sentential complement arguments}

Sentential complement arguments are  always extraposed to the position 
bearing the relation {\em  extraposrel} (unless they are subject to Verb-
raising). Depending on the verbpattern the pronoun {\em het} occurs as an
object. If the verbpattern of the verb contains the specification `HET'
(as in syn{\em HET}THATSENT, syn{\em HET}OPENOMTESENT) {\em  het} occurs,
if the verbpattern does not contain such a specification no {\em  het}
occurs. {\bf Notice that this object {\em  het} is realized as a subject
of verbs with thetavp=vp010 or vp012!}. If no {\em het} occurs for
verbs with thetavp =vp010, vp012, the `subject position' will contain 
{\em  er} in the surface

Some examples:

\begin{description}
  \item[vp120, het] Hij betreurt objrel/{\em  het}  extraposrel/[dat hij ziek is] 
  \item[vp120, no het] Hij vindt extraposrel/[dat hij gelijk heeft]
  \item[vp012, het] subjrel/{\em Het } lijkt mij extraposrel/[dat hij gelijk 
heeft]
  \item[vp010, no het] {\em Er} is gebleken extraposrel/[dat hij ziek was]
  \item[vp010, het] {\em  Het} schijnt dat hij gelijk heeft
\end{description}


\subsection{Sentential Prepositional Object Arguments}

Sentential prepositional arguments are always extraposed. They leave {\em  het}
as an object of the preposition behind. This {\em het} will be turned into
{\em  er} by the normal rules. Examples:

\begin{description}
  \item[] Hij rekent {\em  er}op extraposrel/[dat hij mag komen]
  \item[] Hij gaat {\em  er}van uit extraposrel/[dat dit mogelijk is]
\end{description}

\section{Prep en Part}

In structuren van de vorm ...Prep NP P V... of ...NP P V... moet
bepaald worden of P een partikel is of een achterzetsel/tweede deel van een
omzetsel.

\subsection{Strukturen van het type ...Prep NP P V...}

De volgende voorbeelden zijn typerend. Onder 1 staan gevallen waarin P
geanalyseerd moet worden als een partikel; onder 2 staan gevallen waarin P 
geanalyseerd moet worden als het tweede deel van een omzetsel

\begin{tabular}{|c|c|c|c|c|}
\hline
\# & Prep  & NP        & P    & V\\
\hline
\hline
  & voor  & een boeman   & door & gaan\\
1 & tegen & de reis      & op   & zien\\
  & op    & z'n intuitie & af   & gaan\\
  & tegen & de jeugd     & uit  & varen\\
\hline
  & naar  & Amsterdam & toe  & gaan\\
  & onder & de brug   & door & lopen\\
2 & tegen & de berg   & op   & klimmen\\
  & op    & de kust   & af   & gaan\\
  & tegen & de kade   & aan  & varen\\
\hline
\end{tabular}

De volgende criteria kunnen worden gebruikt om vast te stellen of P een
partikel is of het tweede deel van een omzetsel:

\begin{description}
  \item[Omzetsel(sterk)] Indien de combinatie Prep...P geen omzetsel van het Nederlands
                 is, dan is P een partikel. Dit is een hard criterium, maar 
                   meestal heb je er niet veel aan.
  \item[Path PP(zwak)] Als P het tweede deel van een omzetsel is, dan moet de 
combinatie Prep NP P een `path'-PP zijn (directioneel). Meestal moet de NP dan
een potentiele locatie aangeven (vaak: moet concreet zijn). Het werkwoord moet
een beweging uitdrukken. Dit is echter geen
absolute eis. Met name kan idiomatisering zijn op getreden.

  \item[PPoverV(sterk)] Indien de volgorde P V Prep NP in bijzinnen mogelijk is, dan
is P een partikel. Indien deze volgorde niet mogelijk is, dan is P het tweede
deel van een achterzetsel.

Vgl. {\em  *dat hij doorging onder de brug; *dat hij toeging 
naar Amsterdam; *dat hij opklom tegen de berg} versus {\em  dat hij afging op 
z'n intuitie; dat hij uitvoer tegen de jeugd; dat hij opzag tegen de reis}

\item[Waar Prep(sterk)] Indien vragen van de vorm {\em Waar Prep ... P V} mogelijk 
zijn, dan is P een partikel. Indien dit niet mogelijk is, dan is P het 
tweede   deel van een omzetsel.

Vgl. {\em  *waaronder is hij doorgegaan; *waarnaar is hij toegegaan;
*waartegen is hij opgeklommen; *waartegen is hij aangevaren} vs. 
{\em  waartegen is hij uitgevaren; waartegen heeft hij opgezien}. Merk ook op 
dat de vraag {\em  Waarop is hij afgegaan} wel beantwoord kan worden met
{\em  op z'n intuitie}, maar niet met {\em  op de kust}.

\item[hulpwerkwoord] Als P het tweede deel van een omzetsel
is, en het werkwoord tweeplaatsig, dan wordt het vervoegd met 
{\em  zijn}, niet met {\em  hebben}. Dus, als het werkwoord tweeplaatsig is
en vervoegd wordt met {\em hebben}, dan is P en partikel.

Vgl. bijv. {\em  Hij heeft de vrachtauto ingereden} (partikel) vs. {\em  hij is de 
vrachtauto ingereden} (achterzetsel)

\item[Contrast(sterk)] in dien een zin gemaakt kan worden van het type
{\em hij V er Prep P, niet Prep P}, dan is P het tweede deel van een 
achterzetsel. Vgl {\em  Hij klom ertegen op, niet  van af} (achterzetsel) vs.
{\em  *Hij zag ertegen op, niet van af} (partikel)

\end{description}

\subsection{Strukturen van de vorm ...NP P V...}

De volgende voorbeelden zijn typerend. Onder 1 staan gevallen waarin P
geanalyseerd moet worden als een partikel; onder 2 staan gevallen waarin P 
geanalyseerd moet worden als het tweede deel van een omzetsel

\begin{tabular}{|c|c|c|c|c|}
\hline
\#  & rest   & NP          & P    & V\\
\hline
\hline
 1 &        & de man      & op   & bellen\\
   &        & de marathon & uit & lopen\\
\hline
   &        & de winkel   & uit  & lopen\\
 2 &        & de berg     & op   & klimmen\\
   & iemand & de winkel   & uit  & werken\\
\hline
\end{tabular}

De volgende criteria kunnen worden gebruikt om vast te stellen of P een
partikel is of een achterzetsel:

\begin{description}
  \item[Path PP(zwak)] Als P een achterzetsel is, dan moet de 
combinatie NP P een `path'-PP zijn (directioneel). Meestal moet de NP dan
een potentiele locatie aangeven (vaak: moet concreet zijn). Het werkwoord moet
een beweging uitdrukken .Dit is echter geen
absolute eis.
  \item[R-relativisatie(sterk)] Indien relativisatie van NP mogelijk is met behulp van
het relatief voornaamwoord {\em  waar}, dan is P een achterzetsel.
Indien relativisatie alleeen mogelijk is met {\em  die} of {\em  dat}, dan
is p een partikel. Vgl. {\em  *de man waar hij opbelde; *de marathon waar hij
uitliep} versus {\em  de winkel waar hij uitliep; de berg waar hij opklom;
de winkel waar hij hem uit werkte}

Evenzo geldt dat de mogelijkheid de NP te vervangen door
{\em  er} inhoudt dat er sp[rake is van een achterzetsel.
Vgl. {\em Piet liep de winkel in, maar Karel liep {\bf er} uit}.

\item[hulpwerkwoord(sterk)] Als P een achterzetsel is, en het werkwoord tweeplaatsig,
dan wordt het vervoegd met {\em  zijn}, niet met {\em  hebben}.
Dus, als het werkwoord tweeplaatsig is
en vervoegd wordt met {\em hebben}, dan is P een partikel. Vgl.
{\em  Hij {\bf heeft} de marathon uitgelopen} (partikel) met
{\em  Hij {\bf is  } de winkel uitgelopen} (achterzetsel)

\item[`Dubbele' objecten(sterk)] Indien bij drieplaatsige werkwoorden structuren 
optreden van de vorm NP$_{1}$ NP$_{2}$ P V waarin de eerste NP een direct object is,
dan is P een een achterzetsel. Voorbeeld: {\em De directeur heeft 
de man(NP$_{1}$) de winkelNP$_{2}$ uit (P) gewerkt(V)}. Je kan testen of 
NP$_{1}$ een direct object is of niet door te kijken of deze NP, vervangen door
{\em  hij} op kan treden als subject bij passivisatie. Indien dit
mogelijk is, dan is deze NP een direct object, anders niet. Vgl.
{\em  Hij werd (door de directeur) de winkel uitgewerkt}. In een zin
als {\em  De directeur gaf de man het touw aan} is {\em  de man} geen direct 
object, zoals blijkt uit de ongrammatikaliteit van {\em  *Hij werd (door de 
directeur) het touw aangegeven} (en {\em  aan} is dan ook een partikel hier). 

\item[Topikalisatie] in de combinatie NP P als geheel vooropgeplaatst kan 
worden, dan is P een achterzetsel. Vgl {\em  De winkel uit loop je niet zo
gemakkelijk} vs {\em  *de marathon uit loop je niet zo gemakkelijj}. 
Dit criterium is zeer sterk, maar de resultaten van topikalisatie zijn m.i.
meestal niet volledig grammatikaal, zodat de beoordeling bemoeilijkt kjan 
worden. Indien
de combinatie NP P niet vooropgeplaatst kan worden, dan weet je nog niets.
\end{description}

specifiek vermeld staat dat deze niet voor een van deze betekenissen geldt.
Voor de betekenissen die een andere synvps/thetavp nemen kunnen de overige

synvps en thetavp
----------------

De status van pp is vaak onduidelijk: argument of modifier. De volgende 
richtlijnen zijn gehanteerd.
- een verplichte pp is altijd argument
- een pp is altijd een argument als het in de doeltaal een np wordt
- een optionele pp met een vaste prep komt meer in aanmerking voor een
  argumentstatus naarmate de vertaling van de prep minder letterlijk is
- een optionele pp die op een of andere manier als instrument kan worden
  opgevat wordt niet als argument beschouwd
- een optionele directionele pp wordt als argument beschouwd
- een optionele localiteit pp wordt niet als argument beschouwd
- naarmate de prep in een pp meer vormen kan aannemen wordt de kans dat die pp
  een argument is kleiner

Wanneer wordt een betekenis gemarkeerd met # ?
- als het om een Zuidnederlandse variant gaat
- als de betekenis fout is
- als de betekenis zeer gemarkeerd is
- als de betekenis onduidelijk is en de VD geen uitsluitsel geeft
- als een betekenis 'dubbelop' is; er wordt dan naar de 'originele' betekenis 
  verwezen
- als de betekenis slaat op een ander werkwoord doordat de VD onterecht een
  partikel aanneemt, of juist als de VD onterecht een partikel weglaat
- als de betekenis van een werkwoord regelmatig wordt gevormd door
  toevoeging van een bijwoord
  Vb:
  door -> 'doorXen', door gaan met X 
  uit -> 'uitXen', tot een einde komen met X
  terug -> 'terugXen', de actie X herhalen t.o.v. van subject vorige keer X
  over -> 'overXen', opnieuw X

Wanneer worden betekenissen als dubbelop beschouwd ?
- als het onderscheid met een andere betekenis van hetzelfde werkwoord miniem
  of niet aanwezig is en de vertaling in het Engels hetzelfde
- als het onderscheid met een andere betekenis van hetzelfde werkwoord miniem
  of niet aanwezig is en de vertaling in het Engels weliswaar verschilt, 
  maar zonder dat er enige motivatie voor dit verschil is en de beide betekenis-
  sen evengoed naar dezelfde vertaling zouden kunnen gaan

Wanneer is iets als (semi-)idioom gemarkeerd ?
- Een betekenis geldt als idioom wanneer zowel werkwoord als argument(en) niet
  letterlijk interpreteerbaar zijn en de vertaling niet letterlijk kan zijn.
  Hoewel deze omschrijving wat vaag is, zijn dit de makkelijk te herkennen
  gevallen.
  Vb:'uit de school klappen'
     'in de lach schieten'
- alle betekenissen die slechts kunnen worden gevormd mbv van een ander ww
  zijn als idioom gemarkeerd
  Vb:'laten zitten'
- constructies met 'het' in 'het uithouden' of 'het winnen van' zijn idioom
- een constructie is een semi-idioom als:
  - binnen een betekenis een verschillend object voor een andere vertaling van
    het werkwoord zorgt
  - als het ww niet letterlijk wordt vertaald
  Dit zijn twee moeilijk te hanteren criteria. Bij het eerste punt kan worden
  opgemerkt dat de algemene strategie is geweest: hoe meer objecten een ww
  in eenzelfde vertaling kan hebben, hoe kleiner de kans dat het semi-idioom
  is.
  Voor wat het tweede punt betreft, bij veel ww is het moeilijk uit te maken
  wat de letterlijke betekenis is. De VD is daarbij geen hulp. Soms kan de
  letterlijke betekenis zelfs gemarkeerder zijn dan een zogenaamd figuurlijke
  betekenis.
  Geconcludeerd mag worden dat deze strategie zeer intuitief is, maar in
  ieder geval die gevallen eruit licht die op grond van de VD toch fout wa-
  ren vertaald, omdat de VD of de verkeerde vertaling geeft, of een vertaling
  geeft die slechts voor een zeer beperkt aantal objecten opgaat.

Wanneer zijn extra betekenissen toegevoegd ?
  Er zijn twee methoden voor toevoeging van extra betekenissen gebruikt:
1) een bestaande betekenis opdelen
2) een nieuwe betekenis toevoegen
  Een betekenis is toegevoegd als dat op grond van verschillende thetavp en
  synvps bij een bepaald gebruik van een ww nodig blijkt. Soms vergeet de VD
  een onderscheid te maken dat uit de voorbeeldzinnen wel degelijk blijkt, soms
  was eigen kennis oorzaak van toevoeging van een nieuwe betekenis.
  Het onderscheid tussen 1) en 2) is niet helder en ook niet echt belangrijk.
  Over het algemeen kan worden gesteld dat voor een deelbetekenis is gekozen
  als een omschrijving van VD deze deelbetekenis dekte, in andere gevallen
  werd een nieuwe betekenis toegevoegd.
  Veel voorkomende deelbetekenissen zijn reflexieve varianten die anders
  vertalen dan hun transitieve tegenhangers. Ook bewegingswerkwoorden zijn
  veelal gesplitst in een intransitieve en een ergatieve (met directionele
  pp) variant.

Middle verbs  
  Middle verbs zijn apart gemarkeerd. Het gaat slechts om een kleine klasse
  werkwoorden met daarbij ook nog eens een aantal dubieuze gevallen. Middle
  vorming lijkt in het Nederlands geen produktief proces.
  In bepaalde domeinen lijken middle verbs meer op hun plaats.
  vb.:'de bal trapt/gooit lekker'
      'dit staal last rottig'
      'deze zin vertaalt eenvoudig'
  Bovenstaande zinnen lijken acceptabeler in resp. de sport-, bouw-
  en vertaalwereld.
  Slechts in die gevallen waar een middle verb algemeen bruikbaar lijkt, is
  het werkwoord gemarkeerd.
  Groter, en wel produktief, is het type constructie dat gemarkeerd is als
  semi-middle. Het aantal gemarkeerde gevallen is in het woordenboek gering,
  omdat het hier niet om argumenten gaat. Vooral werkwoorden die een
  'instrument-rol' uitdelen kunnen deze semi-middle vormen
  vb.:'dit geweer schiet lekker' geweer = instrument
      'de schoenen lopen soepel' schoenen = instrument?
      'de schaar knipt zwaar' schaar = instrument
      'de weg rijdt lekker' weg = lokatie
      'de fiets rijdt lekker' = instrument? of is 'fiets' argument

Psych verbs
  Alle werkwoorden die sententiele subjecten toestaan zijn gemarkeerd als 
  psych, ongeacht de waarde voor thetavp.
  Dus als de testzin 'Het V mij dat ...' mogelijk is, dan geldt het ww
  als psych verb.


Creative verbs
  Werkwoorden die op een of andere manier kunnen worden opgevat als iets 
  creerend zijn gemarkeerd. Dit om twee redenen:
  - deze ww lijken altijd een voorNP te kunnen nemen die in veel gevallen
    in het Engels als een np vertaalt.
    Vb.: 'ik bak voor hem een eitje' 
         'ik bouw voor hem een huis'
  - deze ww lijken altijd een leeg object te kunnen hebben, zonder dat de
    betekenis van het ww. verandert

Result verbs
  Een vrij grote klasse werkwoorden neemt een resultatieve bepaling.
  In eerste instantie werd in overleg met Jan O. de volgende procedure
  gehanteerd:
  -constructies als 'in tweeen bijten', 'in stukken delen' krijgen
  pattern synDONP_OTHEROPENPREPPPROP.
  -constructies als 'hij verft de deur groen' krijgen pattern synDONP_
  OPENADJPPROP
  -constructies als 'in/aan elkaar solderen' zijn semi-idiomatisch.
  Bij nadere beschouwing bleek echter dat de klasse resultatieve ww op deze
  manier vrij groot werd en bovendien een bepaalde klasse resultatieve
  bepalingen neemt.
  Al deze werkwoorden zijn nu gemarkeerd, maar omdat het lijkt alsof hier
  een regelmatig proces in werking is, zijn de later gevonden ww van dit type
  niet meer van bovenstaande patterns voorzien. 

'zich' verbs
  Dit zijn werkwoorden die constructies als 'hij ergert zich dood' kunnen
  vormen. In eerste instantie leek dit een vrije kleine groep ww, maar bij
  nadere beschouwing kun je bij heel veel ww, vooral die die een actie uit-
  drukken een constructie met zich toevoegen.
  Vb.: 'hij roept zich schor', 'hij rookt zich een longaandoening', enz.
  De nu gemarkeerde ww vormen dus maar een klein percentage van deze ww.
  Wel zijn het over het algemeen de 'betere' gevallen.

Quasi-reflexieven
  Werkwoorden die 'zich' ipv 'zichzelf' als object (kunnen) nemen, zijn
  apart gemarkeerd als:
  - de betekenis van het ww gelijk is aan die van het ww met een 'normaal'
    object.
  - de Engelse vertaling een werkwoord met 'oneself' als object oplevert

prepkeys:
Bij werkwoorden met twee voorzetselconstituenten is de meest voorkomende
oppervlaktevolgorde gekozen. Dit komt er bij de meeste ww op neer dat de
prepositie van het argument dat als 'theme' kan worden omschreven de tweede
prepkey vult, zodat deze volgorde als standaard is gekozen.
De eerste prepkey wordt gevuld door de prep van het argument dat vaag als
een soort 'goal' kan worden omschreven
Het gaat hier om een kleine klasse; in totaal zijn er slechts acht gevonden:
aandringen, aankloppen, onderhandelen, zich onderhouden, praten, spreken,

Vb: 'bij iemand (1) aandringen op iets (2)'
    'met iemand (1) twisten over iets (2)'
    'met iemand (1) van iets (2) wisselen'

COMMANDO FILES GEBRUIKT BIJ INVULLEN

De commando files hebben de extensie .edt. De naam slaat in de meeste
gevallen op het attribuut waarvoor het file gebruikt wordt.
In de files staat wat welke knop doet (muv synvps.edt)
De commando files worden binnengehaald door het commando ^E gevolgd door de
naam van het file. Dit kan zowel bij files die met modify als bij files die
met het commando edt worden bewerkt.
Aanwezige commando files:

controller.edt
ipp.edt
perfauxs.edt
possvoices.edt
prepkeys.edt
verbraiser.edt
synvps.edt
thetavp.edt
pattern.edt
splits.edt

pattern.edt bevat knoppen om diverse operaties op bverb files uit te voeren
splits.edt bevat knoppen om nieuwe betekenissen een eigen record te geven

De volgorde waarin de attributen zijn ingevuld is als volgt:
thetavp en synvps (tegelijk)
controller
verbraiser
ipp
perfauxs
prepkeys
possvoices

Voor een goede werking van de knoppen is het noodzakelijk dat controller en
verbraiser pas worden ingevuld als synvps is ingevuld. ipp kan pas worden
ingevuld als verbraiser al is ingevuld.

synvps.edt wordt gebruikt in combinatie met thetavp.edt. Roep eerst met ^E
thetavp.edt op en vervolgens synvps.edt. De omgekeerde volgorde leidt tot
problemen.

Met synvps.edt kan op een makkelijke wijze een keuze worden gemaakt uit de
synvps. Als in een bverb file (de main buffer) synvps.edt wordt aangeroepen
krijgen de volgende knoppen betekenis:
F17: vraagt om de waarde voor thetavp. Als deze wordt ingevuld en afgesloten
     met 'enter' dan komt de gebruiker in een work buffer met daarin alle
     mogelijke synpatterns voor deze waarde van thetavp.
F18: gaat van main buffer naar work buffer (mits reeds gecreeerd)
F20: inserteert de in de work buffer geselecteerde patterns op de juiste plek
     in het record in de main buffer.
In de work buffer krijgen de volgende knoppen betekenis:
F18: gaat van work buffer naar main buffer
F19: vraagt om string letters en deleert al die patterns in de work buffer
     die deze string in zich hebben.
     Vb: het is duidelijk dat betreffend ww geen prepositie neemt, vul dan 
     de string 'prep' in en al de patterns waarin een prepositie voorkomt 
     worden gedeleerd.

WAARSCHUWING: Er zit een fout in synvps.edt. Als via F18 vanuit de work
buffer naar de main buffer wordt gegaan en daar via F20 patterns worden
geinserteerd, bestaat de kans dat de patterns niet in het record waaraan
gewerkt wordt worden geinserteerd, maar in het record erboven. Dit komt
alleen voor als de cursor boven in het record staat waarin gewerkt wordt.
Als de cursor enkele regels onder het begin van dit record staat, is deze
fout uitgesloten.

KRITIEK OP VAN DALE (NE en NN samen)

Opmerkingen over Van Dale met betrekking tot invullen ww-patterns.

- Onderscheid tussen idioom en 'echte' betekenis is zeer vaag. Voorbeeldzinnen
  zijn vaak idiomen. Dat is voor vertalers waarschijnlijk heel handig, voor het
  invullen van een woordenboek buitengewoon onhandig.
- Er is weinig bruikbare informatie over patterns. Met name bij PP's blijft
  status vaak onduidelijk, omdat de VD deze nooit als argument beschouwt.
- Soms staan er verkeerde voorbeeldzinnen bij betekenis, soms zijn voorbeeld-
  zinnen onderling verwisseld.
- De volgorde van betekenissen is in de NE niet altijd hetzelfde al in de NN
- Vaak zijn betekenissen niet van elkaar te onderscheiden, ook niet in
  vertaling.
- De criteria waarop verschillende betekenissen worden onderscheiden zijn
  niet helder en als ze dat wel zijn, niet consequent gehanteerd.
- Werkwoorden worden vaak onterecht als onovergankelijk opgenomen, zelfs als er
  duidelijk overgankelijke voorbeeldzinnen worden gebruikt.
- Middle constructies worden niet altijd aangeduid en als ze wel worden
  opgenomen, is de middle constructie los opgenomen, zijnde een aparte
  betekenis.
- Partikels vormen een probleem in de VD. Soms wordt een ww. onterecht, in een
  bepaalde betekenis, van een partikel voorzien, in andere gevallen ontbreekt
  dit partikel onterecht.
  Vb: 'op iemand toe lopen' is volgens VD ww 'toelopen'.
  Vb: 'een jas dichtknopen' is volgens VD ww 'knopen'
- De scheiding tussen betekenissen met een apart lemma (onderverdeling via Rom.
  cijfers)is weinig consequent:
  - Reflexieve werkwoorden krijgen soms wel, soms niet een eigen lemma.
  - Als een werkwoord een optioneel object heeft, leidt dit soms tot twee 
    aparte lemma's, terwijl in andere gevallen dit gewoon als een werkwoord
    wordt behandeld.
  - Het is niet altijd duidelijk waarom een ww soms via Romeinse cijfers wordt 
    onderverdeeld (en dan in de bverb files een apart lemma krijgt) en in
    andere gevallen slechts meer betekenissen worden gerankschikt onder
    hetzelfde lemma.
- Het komt vaak voor dat een bepaald woord vertaling X krijgt in een van zijn 
  betekenissen, terwijl in de daarbij behorende voorbeeldzinnen allerlei
  vertalingen staan van deze betekenis, behalve X.
- Sommige betekenissen krijgen meerdere vertalingen, waarbij de verschillende
  vertalingen toch een andere betekenis hebben.
