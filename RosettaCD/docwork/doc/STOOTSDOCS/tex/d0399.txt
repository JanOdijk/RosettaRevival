\section{Introduction}
Most of the methods for semantic desambiguation require some kind of
world model. For some methods a taxonomy, a hierarchy of sorts, must
be created, for other methods a knowledge base or a semantic network 
is required. However,
until now these approaches are not very successful ( at least not for
non-toy applications ) and there
are many problem areas which have to be investigated.
A major problem is, as we have stated above,  that one has to
construct a model of the domain of interest. At the moment there are
no theories how such models can be constructed automatically, for instance
from lexicons or corpora. This implies that the creation of these models must
be done by hand. The choice of 
a new application domain implies an enormous
effort in order to create a new model for that specific domain.
Furthermore, these models are never complete, because
it is impossible to put all real world subtleties into a computer.

Recently, some researchers have used statistical methods to analyze huge
corpora. For instance, Hindle's Noun Classifier is "... an approach to 
classifying English words according to the predicate-argument
structures they show in a corpus of text. The general idea is straightforward:
in any natural language there are restrictions on what words
can appear together in the same construction, and in particular
on what can be arguments of predicates. " Another project, which  uses
statistical methods is the YTH Statistical Approach to Machine
Translation. In this project a corpus of 2 million English sentences
and their French translations (the Hansard Corpus) is used.

One could characterize the difference between corpus statistics and 
world modelling as the usage of implicit information versus explicit 
information.
With statistical methods it is possible to use the information which is
implicit in a corpus. For the creation of a world model an explicit definition
of the world entities and their relations is required.

The advantages of statistics are obvious. In general, statistic methods are
robust in contrast with rule based mechanisms which are very fragile.
Furthermore, a statistic method is general (reusable). If a new domain must be
 used
for an application, one can use the same statistic method for the
analysis of a corpus belonging to that domain. In
the next section we'll show, very informally, how statistical methods could 
be used in Rosetta. It is not our goal to propose drastic changes in the
formalism of the Rosetta system. The statistics should be looked at as a 
kind of heuristic measures wich will be {\em added} to the system.

\section{Statistics in Rosetta}
Before introducing the statistical methods we have to define some new terms.
A {\em seb} is  a sequence
of basic expressions  which is composed of the terminals of a syntactic
derivation tree from left to right . The term {\em smk}  is used
for the sequence of meaningkeys which is composed of the terminals of
an IL-tree.

Statistical methods can only be successful if a huge amount of data is
 available.
For the purpose we have in mind this must be a large machine readable corpus 
preferably about one  topic, e.g. information technology, computers etc..
Besides the fact that such a corpus can be used for statistic analysis, it
it also, at least in our opinion, a good way to pinpoint our application
domain. Our goal would then be "to be able to analyse and translate all 
sentences occuring in the corpus". It seems to us an excellent way to extend and
adapt our grammar in order to cover "real world" sentences that occur
in a particular domain.

\subsection{Corpus building}
The corpus which was mentioned above,  is not the one we finally need. The
corpus to which we apply our statistic methods consist of pairs  $(seb, smk)$.
It has to be generated from the original corpus using the analysis part of
the Rosetta
system in an interactive mode. In this mode the (professional) user
selects the best syntactic derivation tree and for that derivation tree
the best semantic derivation tree for each input sentence. These choices will
then be added to a database of $(seb, smk)$ pairs. Ultimately, this database
will be the corpus we are going to use.

\subsection{Statistical methods}

The first statistical method we have in mind is the following. If we apply
Rosetta to an input sentence  the M-Parser will yield a number of possible
syntactic derivation trees for that sentence. We assume that 
structural differences between these trees are reflected in the {\em sbe's}.
We would now like to have a statistical method which says which {\em sbe}
of a set of {\em sbe's} is the most probable one. In order to find it we must be
able to calculate the probability $P(sbe_{i})$: the probability that 
$sbe_{i}$ is an element of the (artificial) $sbe$ language. This is possible
using the trigram model that is also often used in speech recognition.
This trigram model of the language under consideration is constructed
using  only the $seb$ sentences from the corpus of $(seb, smk)$ pairs.

The second statistical method is based on the concept of "Mutual Information",
a concept which originates from the field of
Information Theory. Mutual information is a quantitative measure of how
much the occurrence of a particular mkey $a$ in an $smk$ tells us about the 
possibility of another mkey $b$ in the $smk$. It is defined as
\[ I(b;a) = \log(P(b\mid a)/P(b)) \]. We will use it for making a choice
from the set of $smk's$ which are possible translations of the $seb$ which 
was the most probable  syntactic derivation tree bottom. We can
write an $seb$ as $(s_{1},\ldots,s_{n})$. With each $s_{i}$ a set $M_{i}$ of
possible mkey's corresponds. Our task
is now to pick from each set one mkey and to construct in this way
the "best" sequence of mkeys.  We can define the mutual information
of an $smk$ $(m_{1},....,m_{k})$ as
\[ \Sigma_{i=1}^{k}\Sigma_{j=1}^{k} I(m_{i}; m_{j}) \], where $I(m_{i}; m_{j})$
is the mutual information between $m_{i}$ and $m_{j}$. 

The most likely candidate for the  "best" mkey sequence is  the sequence which
containes the highest mutual information. We realize that the search space
of this problem can be very large, viz. the cartesian product 
$M_{1}\times\ldots\times M_{k}$.

\section{Conclusions}
As we pointed out in the previous sections, adding the methods described
in this paper to the Rosetta system will not yield a completely different 
system. Statistical methods will only be used to shrink the set of ambigous
syntactic or semantic derivation trees. The system can  function
perfectly without  statistics. Then, however, it will generate all
possible ambiguities.
It should be clear that the statistics can only be collected if
the grammar and the lexicons cover most of the sentences in the
corpus. In fact the best action sequence is ( in our opinion):
\begin{itemize}
\item Creation of a grammar and a lexicon such that most of the sentences
      of the corpus are accepted by the Rosetta analysis component.
\item Creation of the $(sbe, smk)$ corpus, by applying the Rosetta
      analysis component to each of the sentences in the corpus.
\item Extraction of statistic data from the $(sbe, smk)$ corpus, by means
      of general algorithms.
\item Usage of the collected statistical data by the Rosetta system.
\end{itemize}
In order to gain a clear insight in the power of the proposed  statistical 
methods,
it would be wise to do a  feasability study  in which a large corpus is used. 
This 
can be an automatically generated corpus to which preferences for certain
"language constructs" have been added. Furthermore, some methods have to be
developed, such as the trigram model and the search algorithm for finding
the optimal mutual information.
