\section{Introductory Remarks}

Some notational Conventions:
\begin{description}
   \item[names of rule classes] are prefixed by {\em RC:}
   \item[names of rules] are prefixed by {\em R:}
   \item[names of transformation classes] are prefixed by {\em TC:}
   \item[names of transformations] are prefixed by {\em T:}
   \item[names of Filter Classes] are prefixed by {\em FC:}
\end{description}


This document only defines subgrammars. It does not give a specification of
the members of Rule and Transformation classes. For this, see the documents,
0153 and 0155.
For each subgrammar the HEADCATS, IMPORTCATS, EXPORTCATS and the CONTROL
EXPRESSION is given. Furthermore, the 'ESSENTIAL RULE(S)' have been indicated,
i.e. the rule(s) that are applicable if any rule or transformation from the
subgrammar is applicable. (cf. doc 101, p.??). These rules occur in any
instantiation of the control expression.

The control expressions and essential rules are not given in terms of
rules, but in terms of Rule Classes.

After each class in the control expressions either {\em (E) } or {\em (C) }
has been added. {\em (E) } indicates that the rule class has been described
in doc. 0153; {\em (C) } indicates that such a description can be found in
doc. 0155.

In accordance with doc. 101, we have 
changed certain category names to satisfy the requirement that the
intersection of HEADCATs and EXPORTCATS is the null set.
For the major categories (BNOUN, BVERB, BADJ, BADV) the following
derivation stages are run through, if no sentences are formed
on basis of them, where X ranges over these major categories:


\begin{enumerate}
  \item[]  BX
  \item[]  SUBX
  \item[]  XPPROP
  \item[]  XPFORMULA
  \item[] \{OPENXPPROP, CLOSEDXPPROP \}
\end{enumerate}

PREPs go through the same stages, though no categorial distinction between
B and SUB is made.

The change from XPPROP to XPFORMULA is made in the RC:Clauseformationrules;
The change from XPFORMULA to OPENXPPROP or CLOSEDXPPROP is made in the
RC:Mooddetermination.

Furthermore, as announced in doc 0146 on domain T, the categories 
VERBPFORMULA, OPENVERBPPROP, CLOSEDVERBPPROP exist as well.



This document contains no specification of the subgrammars for
NP's and  NPPROP's, ADVPs and ADVPPROPs. These have been/ will be specified
by Franciska de Jong.
The rules and transformations for time are not specified here either. This will
be done by Lisette Appelo. This document does indicate where that part must be
incorporated here.

A special section is devoted to the subgrammars ADVPformation, ADVPtoADVPFORM
and ADVPFORMtoADVP. These are subgrammars that are equal to the
ADVPPROPformation, ADVPPROPtoADVPFORMULA and ADVPFORMULAtoADVPPROP subgrammars
respectively (although they will contain other rules). These subgrammars
are required for the treatment of adverbials by means of abstract prepositions.
They are somewhat special, and do not fit nicely into the general organization,
but that is the cost of treating (certain) adverbs by means of abstract 
prepositions.

\section{ Derivation Subgrammars}

\subsection{General Remarks}

The subgrammars described in this section account for derivation. They
turn a BX into a SUBX.
It is assumed that suffixes (and prefixes, but it happens almost never to be
the case) can be the head.
There is a default rule for 'derivations' of PREP's. This is to retain the
possibility of translating between PREPs and other major basic categories
(BADJ, BADV etc.). 

I have assumed the existence of the {\em suffix categories} BADJSUFF,
BADVSUFF, BNOUNSUFF, BVERBSUFF, in accordance with the assumptions
made by Agnes Mijnhout. I assume that these categories have BXrecords,
i.e. BADJrecord, BADVrecord, BNOUNrecord, BVERBrecord.

An alternative would be to assume no special 
categories, but a special attribute in BXrecords: 
\begin{itemize}
  \item morphemestatus:morphemestatustype:free
\end{itemize}
where {\em morphemestatustype } is defined as:
\begin{itemize}
  \item morphemestatustype = (bound, free);
\end{itemize}

If the value of this attribute equals {\em free}, then it is a word, otherwise
an affix.

I have assumed that only SUBXs (and not BXs) can be imported in these
subgrammars. The reasons is that in this way a superfluous ambiguity
can be avoided. (i.e. SUBX-s must be imported anyway, and now BX's are 
excluded). It does mean that
certain rules must be formulated differently than in the morphology. e.g.
{\em ly }(BADVSUFF) + {\em  interesting} (SUBADJ[head/BADJ]) instead
of {\em ly }(BADVSUFF) + {\em  interesting} (BADJ).

The control expressions consist of one rule class turning a BX into a SUBX,
and a rule class turning a BXSUFF plus a SUBY into a SUBX (X,Y ranging over
ADJ,ADV,NOUN, VERB).
To illustrate we give the derivation for the adverb {\em interestingly}:

\begin{enumerate}
  \item[] BADJ(interesting)          $\rightarrow$ (by BXtoSUBX)
  \item[] SUBADJ[head/BADJ(interesting)] + BADVSUFF  $\rightarrow$ (derivation)
  \item[] SUBADV[compl/BADJ(interesting), head/BADVSUFF(ly)]
\end{enumerate}

is not in accordance with the more restricted variant of control expressions.
See also doc 101.

\item Crucial ordering:
      \begin{itemize}
	\item Nowhshiftsubstitution  $\prec$ Shiftrules  $\prec$
Shiftsubstitution. This makes  it possible to extract parts of a constituent by
shifting, as is necessary in   such cases as {\em Who did you see a picture of }
      \end{itemize}

\end{itemize}
