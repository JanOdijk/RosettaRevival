\begin{description}
\item[Datum] 21 september 1987
\item[notulist] Carel Fellinger
\item[Aanwezig] Henk Dijkstra, Carel Fellinger, Maarte van Hattem, Leen Helmink,
                Hans Oerlemans

\item[Agenda]
   \begin{enumerate}
   \item organisatorische zaken
   \item inventarisatie van bestaande en/of op het lab. aanwezige en/of
         benodigde tools
   \item formulering doestelling tools groep
   \item opstellen activiteiten plan, eventuele taakverdeling
   \item rondvraag en sluiting
   \end{enumerate}
\end{description}


\section{organisatorische zaken}
Besloten werd in het vervolg notulen bij te houden van de tools-groep 
vergaderingen. Deze notulen zullen onder het Rosetta document beheerssysteem
bewaard worden. Daartoe moeten notulen naar Carel Fellinger gestuurd worden via 
het email adres prle::nl150::fellinger. De notulen zullen naar de leden van de 
toolsgroep gestuurd worden alsmede de project en groep leiders, te weten:
H Dijkstra, C Fellinger, M van Hattem, L Helmink, ACM Oerlemans,
HH Eggenhuisen, EAM Odijk, van Roermund (WY8), HBM Jonkers, WJ van Gils,
RW Schutten, RMC Ahn, AJ Nijman.

De vergaderfrequentie is gesteld op eens per twee maanden. De eerst volgende 
maal zal echter binnen twee maanden zijn om een snelle inventarisatie mogelijk 
te maken.


\section{inventarisatie van bestaande en/of op het lab. aanwezige en/of benodigde 
tools}
De volgende computers zijn in onze groepen aanwezig: VAX/VMS, VAX/Ultrix,
Sun, Apollo, Transputer, Symbolics en binnenkort Atari ST.

De volgende programmertalen zijn beschikbaar: Fortran, C, Pascal, Modula2, 
PoolT, occam (een parallelle taal), common-lisp, prolog en
ML (een functionele taal).

Verder zijn aanwezig: de amsterdam compiler kit, het volledig TeX pakket (dus
TeX, LaTeX, BibTeX, SliTeX, tangle en weave), de NAG bibliotheek (een 
verzameling mathematische functies), een TeX previewer (enkel op de sun), het 
operating systeem Xinu en de compiler generator GAG.

Hier volgt een lijstje van de tools die de verschillende projecten gebruiken:
\begin{description}
\item[Rosetta] werken op VAX onder VMS\\
   \begin{itemize}
   \item RBS, Rosetta beheerssysteem, een laag boven MMS/make en CMS/rcs.
         gedeeltelijk in DCL (dec command language) en in Modula2.
   \item VMS/Pascal en Logitex/Modula2 compiler
   \item Pascal programming standards
   \item een performance tool van DEC
   \item eigen compiler generatoren, gebaseerd op de eerste twee componenten 
         van Rosetta, accepteert geattribueerde reguliere expressies.
   \item aangepaste editors (setup files voor VAX/VMS EDT), voor elk file type
         zijn wat voor dat file type handige extra commando's gemaakt vooral om
         het gebruik van (programming) standards te ondersteunen.
   \item documentatie beheerssysteem
   \item LaTeX
   \item aangepaste LaTeX document style, eigen titel pagina met Philips embleem,
         hoofdzakelijk een copy van de doom/prisma titel pagina.
   \end{itemize}
\item[doom/prisma] werken op Suns onder Unix en op Transputers\\
   \begin{itemize}
   \item PDMS, voorstel voor een eenvoudige versie van RBS
   \item Elegant, een LL(1) compiler generator, genereert Modula2, draait op de 
         Sun
   \item Sun/Modula2 of ACE/Modula2 en Sun/C en ACE/C (cross-)compilers
   \item C programming standards, deze zijn opgesteld door het CST.
   \item Unix standards, X-Open
   \item gprof, een pretty printer
   \item specifieke simulatoren
   \item documentatie beheerssysteem
   \item LaTeX
   \item aangepaste LaTeX document style, eigen titel pagina met Philips embleem
   \end{itemize}
\item[vsp] werken op VAX onder VMS, op Apollo's onder Unix en op IBM\\
   \begin{itemize}
   \item DSEE, Apollo's versie van make
   \item Pascal an C compilers
   \item Pascal preprocessor van Krein Kuipers. Dit omvat een pakket macros die
         voor portabiliteit tussen de verschillende in gebruik zijnde Pascal
         compilers zorgt, ook voor niet standaard Pascal zaken als separate
         compilatie
   \item Apollo debugger
   \item specifieke simulatoren voor de te maken machine
   \item compiler en interpreter voor de te maken machine
   \item LaTeX en doced, doced is een wysiwyg-editor die faciliteiten biedt om 
         plaatjes te tekenen.
   \item VSPNotes, zorgt voor documenten beheer
   \item GPR en dialogue, tekenpakket en menu handler
   \end{itemize}
\item[genesis] werken op Sun onder Unix, op VAX onder VMS en op Symbolics\\
   \begin{itemize}
   \item de programmeer talen prolog (voor rapid prototyping), Common-Lisp en
         ML, prolog en ML zijn via een esprit partner beschikbaar gekomen en
         derhalve beperkt beschikbaar.
   \item GAG, een compiler generator
   \item phill, voor de symbollics, VAX/VMS en sun. Heeft op de sun 8M nodig!
   \item genesis, een prototype is af, dit is enkel geschikt als demo model.
   \end{itemize}
\item[anderen] werken op VAX onder VMS\\
   \begin{itemize}
   \item C
   \item Amsterdam Compiler Kit
   \item Xinu
   \end{itemize}
\end{description}

Tijdens de vergadering kwamen de volgende wensen op tafel:
\begin{itemize}
\item faciliteiten om plaatjes te tekenen en deze in LaTeX te 'includen'.
\item een simulator generator
\item WEB nader bekijken, aanpassen ann Modula2 , speciale editors voor WEB 
files.
\end{itemize}

\section{formulering doestelling tools groep}
Na enig beraad leek het volgende lijstje een adequate omschrijving van de 
doelstellingen van de toolsgroep:
\begin{itemize}
\item inventarisatie van gebruikte en gewenste tools
\item voorkomen van dubbel werk wat betreft tools
\item standaardisatie en verhogen van de uitwisselbaarheid van door ons zelf 
ontwikkelde tools
\item vraagbaakfunctie wat betreft tools
\end{itemize}
Het begrip {\it tools} dient vrij ruim geinterpreteerd te worden: nl. alle 
programmatuur die het ontwikkelen van software ondersteunt. Hieronder vallen
dus structure-editors, version-control systemen, maar ook aanpassingen aan 
LaTeX om titel pagina's met Philips embleem te maken.

\section{opstellen activiteiten plan, eventuele taakverdeling}
Om een goede inventarisatie van de in gebruik zijnde tools en ervaringen 
daarmee te krijgen wordt een enquete gehouden.
Afgesproken is dat Carel een lijstje van de aan ons bekende tools opstuurt naar
Hans. Deze maakt er een enquete van, waarin verzocht wordt vergeten tools te 
vermelden en ervaringsgraad aan te geven. Carel, Leen, Henk en Hans zullen voor
verspreiding van de enquete binnen de eigen groepen zorgen.

Hans Oerlemans heeft op zich genomen om de vraagbaak functie te bemannen. 
Vragen kunnen enkel met email gesteld worden. Het email adres van Hans is
prle2::oerleman.

\section{rondvraag en sluiting}
Het CST is ook geinteresseerd in beheerssystemen, en willen er zelf een maken.
Deze zal onder VMS en Unix moeten draaien en dient gekoppeld te worden aan 
databases.

Leen biedt aan om een demonstratie van PHIL op Sun en VMS/VAX te geven.

De volgende vergadering, met PHIL demo, is gepland op 26 oktober, 13 uur 30 bij 
Hans op de kamer.
